# 灵枢笔录长期留存与网状玩法优化策划案

> 目标:从专业游戏策划角度,把《灵枢笔录》从“AI 修仙 MVP”推进成一个能让用户持续回来的长线产品。  
> 更新时间:2026-05-24  
> 项目目录:`/Users/bobdong/项目/LingshuCodex`

---

## 1. 总结结论

《灵枢笔录》最独特的卖点不是“修仙文字 RPG”,也不是“战斗时调用 LLM”,而是:

> **玩家自己的 API Key 是灵脉,Token 是灵气,模型档位是境界,真实 AI 公司是修真宗派。**

这套隐喻非常强,但当前项目还处在“系统很多,闭环不够强”的阶段。代码里已经有不少好雏形:

- 5 门派、9 境界、模型梯度。
- 8 属性、门派友好度、拜入机缘。
- 战斗 v2:数值/叙事解耦、预选队列、跳过叙事、AI HUD、战后章节。
- 探索地图、怪物危险度、60 秒奇遇、赠礼安抚、打坐回血。
- 113 怪物、21 Boss、4 条故事线、55 物品。

但这些系统目前还没有形成强网状结构。玩家的体验容易变成:

```text
接入 Key → 选门派 → 地图点怪 → 战斗 → 掉落 → 回主城
```

这个循环能证明概念,但不足以支撑长期留存。下一阶段最重要的是把它升级成:

```text
今日目标 → 选择修行路径 → 战斗/奇遇/收集 → 材料用途 → 宗门关系变化
      ↓              ↓              ↓             ↓
  周常推进       Boss 线索       修行录收藏     境界突破/模型升档
      ↓              ↓              ↓             ↓
          山海图节点解锁 → 故事线推进 → 赛季目标/轮回继承
```

一句话:

> **不要只增加内容数量,要让每一次战斗都改变玩家在“宗门、怪物、物品、故事线、角色养成”这张网中的位置。**

## 2. 市面上的长线游戏怎么做

下面不是照抄竞品,而是抽取成熟设计方法,再翻译成《灵枢笔录》的修仙语境。

| 市场做法 | 典型产品 | 玩家为什么会回来 | 灵枢笔录可转译 |
|---|---|---|---|
| 体力/资源自然恢复 | 原神 Resin、星穹铁道 Trailblaze Power | “不想浪费上限”,每天上线消耗资源 | 灵气/神识/闭关时间自然恢复,超上限转为低效“散逸灵气” |

| 战令/赛季旅程 | Diablo IV Season Ranks / Battle Pass | 目标清单带动多玩法参与 | “修行纪行”:完成猎妖、炼器、问道、Boss 目标拿外观/称号 |

| Roguelite 局内构筑 | Hades、星穹铁道模拟宇宙 | 每局都有不同祝福组合,重复也新鲜 | “秘境祝福”:剑意、机关、魔念、天命词条随机组合 |

## 3. 设计原则

### 3.1 不做“强迫成瘾”,做“正向习惯”

可以设计让用户持续回来,但不要靠焦虑、惩罚、红点轰炸、无限签到绑架。推荐原则:

- 每天 10 分钟有收获。
- 周末 1 小时有深度。
- 不上线不会毁号,只是少拿部分收益。
- 消耗 API Key 的地方必须透明、可控、可跳过。
- 随机奖励要有保底,不能让玩家觉得被系统戏弄。

### 3.2 每个系统都要接到至少两个系统

孤立系统很容易被玩家遗忘。网状系统的原则是:

```text
战斗不只给经验,还给材料、族群仇恨、宗门声望、Boss 线索、修行录素材。
物品不只是收藏,还要能炼丹、炼器、赠礼、突破、解锁秘境。
Boss 不只是百科,还要推进故事线、改变宗门关系、开放地图节点。
奇遇不只是弹窗,还要改变属性、友好度、地图、后续事件概率。
```

### 3.3 AI 不是装饰,而是“稀缺仪式”

普通刷怪不要每次都让顶级模型写长文。应当分层:

- 高频行为:模板 + Haiku 短句 + 缓存。
- 中频高光:Sonnet 写暴击、Boss 技能、关键奇遇。
- 低频仪式:Opus 写突破、首杀、战后章节、转世回忆。

这样顶级模型的出现才有价值,玩家也会觉得“这次值得花灵气”。

## 4. 目标用户拆解

| 用户类型 | 他们想要什么 | 当前满足度 | 应补的机制 |
|---|---|---|---|
| 修仙小说党 | 世界观、章节、门派恩怨 | 中 | 修行录、Boss 章节、关系图、主线推进 |
| 数值优化党 | 构筑、效率、装备、路线 | 中低 | 技能树、装备词条、山海图刷图专精 |
| 收集党 | 图鉴、稀有掉落、全收集 | 中 | 山海经、材料用途、套装、成就 |
| 轻度上班族 | 每天短时间成长 | 低 | 日课、离线收益、自动修行、预算控制 |
| AI/技术玩家 | 看模型差异、成本、Prompt | 高潜力 | Token 报表、模型对比、功法 prompt 系统 |
| 社交炫耀党 | 分享战报、排行、称号 | 低 | 修行录海报、首杀碑、宗门榜 |

当前最大缺口是轻度上班族、收集党和社交炫耀党。只靠战斗和文字叙事,长期会更像 demo。必须把“每天回来”设计出来。

## 5. 核心留存循环设计

### 5.1 30 秒循环:即时爽点

目标:点一下就有反馈。

```text
选敌 / 点招 → 数值飞出 → 敌我状态变化 → 掉落/战报提示
```

已实现基础:战斗 v2 已有即时数值、队列、跳过。  
优化:

- 卡牌上直接显示预计伤害、命中率、暴击率、灵气后余额。
- 普通攻击必须 2 秒内完成双方数值反馈。
- 回合结束给“下一步推荐”:补血、继续、赠礼、撤退、挑战更强敌。

### 5.2 3 分钟循环:一次修行

目标:一小局有明确开始和结束。

```text
进入地图 → 选一个目标怪/奇遇 → 战斗/赠礼/打坐 → 拿奖励 → 回主城整理
```

优化:

- 探索地图给“推荐目标”:稳胜怪、稀有掉落怪、宗门悬赏怪。
- 地图上每只怪都标注“本次击败可推进什么”:材料、族谱、任务、Boss 线索。
- 打完后不要只返回地图,要给“继续刷同族 / 去炼丹 / 看新解锁故事”。

### 5.3 10 分钟循环:日常闭环

目标:每天上线 10 分钟能完成日课。

建议新增“今日修行令”:

| 活跃点 | 行为 |
|---|---|
| 20 | 击败任意怪物 1 次 |
| 20 | 完成一次打坐 |
| 20 | 触发或处理一次奇遇 |
| 20 | 使用/炼制/赠送一个物品 |
| 20 | 阅读或收藏一段修行录 |

满 100 给今日奖励。玩家不需要做固定任务,只要玩自己喜欢的内容也能完成。这是市面日常系统的成熟做法:给目标,但不要强迫单一路径。

### 5.4 1 周循环:周目标

建议新增“宗门周课”,三选二即可完成:

- 猎妖:击败指定族群,推进山海经。
- 炼器:收集材料,炼成一个法宝/丹药。
- 问道:完成一次 Boss 情报、奇遇链或修行录章节。

周奖励不只是资源,还给“宗门贡献”和“山海图点数”。这样周常能接入长期系统。

### 5.5 6-8 周循环:赛季/纪元

建议新增“天劫纪元”:

- 每 6-8 周一个主题:狐妖乱世、龙渊潮起、异域裂隙、开源觉醒。
- 赛季内开放专属奇遇、地图节点、Boss 支线、头像框/称号。
- 赛季结束保留修行录、称号、部分道基;战力资源部分重置或转入常驻。

市场上 Diablo IV 和大量 GaaS 通过赛季旅程让玩家反复回流。灵枢不一定要强商业化,但可以用“纪元”更新内容节奏。

## 6. 网状系统设计

### 6.1 宗门关系网

当前基础:

- `character.factions` 已存在。
- Home 已展示五派友好度。
- Boss 真实公司宗派与羁绊已经有内容。

问题:

- 友好度目前展示强,用途弱。
- 门派关系没有影响可玩内容。

优化:

| 机制 | 说明 | 为什么 |
|---|---|---|
| 宗门声望商店 | 不同友好度解锁不同物品/心法/情报 | 声望有明确用途 |
| 宗门冲突任务 | 帮沧澜打魔修会降某派好感,升沧澜贡献 | 让选择有代价 |
| 叛徒线 | 墨魔、齿轮鬼等 Boss 作为宗门剧情节点 | 把 Boss 名录变成主线 |
| 中立调停 | 高“缘”属性可调停两派,获得稀有称号 | 让非战斗属性有价值 |

市场做法:

- RPG 常用阵营声望控制商店、任务和剧情分支。
- Diablo/赛季旅程常用任务清单引导玩家体验不同系统。

落地文件:

- 后端:`attributes.py`、`events.py`、`main.py` 新增声望用途 API。
- 前端:`Home.vue` 友好度条加点击详情,`Bosses.vue` 加宗门关系图。

### 6.2 怪物生态网

当前基础:

- 12 族怪物、食物链、行为模式在 `behaviors.py` 已有。
- 探索地图有移动和危险度。

问题:

- 怪物之间的生态关系主要停留在配置,玩家感知不强。
- 击败某族不会影响地图生态。

优化:

| 机制 | 说明 | 为什么 |
|---|---|---|
| 山海经图鉴 | 每族一页:族谱、首杀、掉落、天敌、故事 | 收集党目标 |
| 族群仇恨 | 连杀狐妖会让狐族精英追杀,猛兽族好感变化 | 行为造成后果 |
| 生态事件 | 猛兽追狐、鬼族夜袭、龙族巡天在地图上可见 | 地图活起来 |
| 族群研究等级 | 击败/赠礼/观察同族提升研究等级,解锁弱点 | 不只靠杀 |

市场做法:

- 怪物猎人/宝可梦式图鉴让“遇见内容”变成“收集进度”。
- AFK/放置游戏常用阵营克制和图鉴加成推动长期收集。

落地文件:

- 后端新增 `beastiary` 存档字段。
- `ExploreMap.vue` 怪物详情展示“已研究等级/弱点/掉落缺口”。
- `Bosses.vue` 可拆出 `Bestiary.vue`。

### 6.3 物品炼制网

当前基础:

- `items.py` 有 55 种材料/丹药/法宝/心法/灵宝。
- 背包与物品全集存在。

问题:

- 材料掉了以后用途弱。
- 法宝装备与合成闭环不够。

优化:

| 机制 | 说明 | 为什么 |
|---|---|---|
| 丹方/器方 | 材料组合炼丹炼器 | 让掉落变成目标 |
| 套装词条 | 狐妖套、龙鳞套、鬼修套 | 收集驱动 |
| 词条洗炼 | 消耗材料重铸法宝副属性 | 长线资源黑洞 |
| 赠礼偏好 | 不同怪/宗门喜欢不同材料 | 背包物品多用途 |
| 材料追踪 | 选中目标法宝后地图高亮掉落来源 | 降低迷茫 |

市场做法:

- ARPG/刷宝游戏靠装备词条和材料消耗做长期目标。
- 体力制 RPG 通常让日常材料副本服务角色成长。

落地文件:

- 新增 `recipes.py`。
- `Inventory.vue` 增加“可合成/用途/追踪”。
- `ExploreMap.vue` 高亮被追踪材料来源怪。

### 6.4 山海图终局网

当前基础:

- 探索地图是随机刷怪。
- Boss 有故事线,但和地图关系弱。

问题:

- 地图不是长期进度,刷新后没有“推进感”。

优化:

新增“山海图”,借鉴 Path of Exile Atlas:

```text
终南山入口
 ├─ 狐妖林
 │   ├─ 三尾洞府
 │   └─ 九尾遗迹 → 九尾天狐线索
 ├─ 龙渊
 │   ├─ 蛟龙潭
 │   └─ 真龙殿 → 应龙正神线索
 ├─ 鬼门关
 │   ├─ 白无常路
 │   └─ 阎罗殿
 └─ 异域裂隙 → 界外神祇线
```

每个节点有:

- 主族群。
- 掉落偏好。
- 危险等级。
- 首通奖励。
- 可投入山海图点数增强收益或难度。

为什么:

- 玩家不再是“随机点怪”,而是在拓图。
- 每个节点都能成为长期目标。
- 高玩可以选择专精刷法,轻度玩家按推荐路线走。

市场做法:

- Path of Exile 的 Atlas 把终局刷图做成可选择、可专精、可扩展的网状地图。
- 星穹铁道模拟宇宙用祝福/奇物让重复挑战有局内变化。

落地文件:

- 新增 `map_nodes.py`、`Map.vue` 或重构 `ExploreMap.vue`。
- `battle/start` 接收 `node_id`,影响怪物池和掉落。

### 6.5 修行录与收藏网

当前基础:

- 战斗后章节已经有后端 task 和前端 chapter 面板。
- Home 有 battle_history 雏形。

问题:

- 章节没有成为长期收藏资产。

优化:

| 机制 | 说明 | 为什么 |
|---|---|---|
| 修行录书架 | 保存首杀、突破、奇遇、战败名场面 | LLM 内容不浪费 |
| 自动标题 | LLM 给章节生成标题和标签 | 可检索、可分享 |
| 名句摘录 | 暴击/天命句子可收藏 | 传播点 |
| 战报海报 | 一键生成图片/卡片 | 社交分享 |
| 回忆加成 | 收藏达到数量给称号/小属性 | 收藏有收益 |

市场做法:

- 社交传播型游戏常把高光时刻做成可分享内容。
- Roguelite 和卡牌游戏常保存 run history,方便复盘。

落地文件:

- `store.py` 当前内存存储必须先持久化。
- 新增 `journal.py` / `/api/journal/*`。
- 前端新增 `Journal.vue`。

### 6.6 奇遇因果网

当前基础:

- `events.py` 已有 12 类随机事件,受 HP、疲劳、fate、factions 影响。
- `ExploreMap.vue` 60 秒触发奇遇。

问题:

- 奇遇是点状弹窗,不是链式故事。

优化:

| 机制 | 说明 |
|---|---|
| 奇遇链 | 同一 NPC/怪物/宗门可连续出现 3-5 次 |
| 因果标记 | 玩家选择留下“欠狐族人情”“救过器灵”等 tag |
| 后续回收 | 未来战斗/Boss/商店引用 tag |
| 失败分支 | 拒绝、逃跑、赠礼失败也进入不同后续 |

为什么:

- LLM 最适合写“记住你做过什么”的个性化内容。
- 玩家会觉得世界在回应自己,这是传统数值游戏很难做到的差异点。

市场做法:

- 开放世界和 CRPG 常用任务 flag 让世界状态变化。
- AI NPC 产品强调记忆与个性化,灵枢可以把它数值化。

### 6.7 道基/轮回网

当前基础:

- 有“转世重修”,但更像删档。
- 有 8 属性、拜入机缘、等级成长。

问题:

- 失败、转世、战败的长期价值弱。

优化:

| 机制 | 说明 | 为什么 |
|---|---|---|
| 道基点 | 升境、首杀、章节收藏给道基 | 永久成长 |
| 轮回继承 | 转世保留称号、图鉴、少量道基 | 删档不痛 |
| 心魔试炼 | 战败积累心魔,清除后得属性 | 失败有意义 |
| 多流派开局 | 转世时可选前世遗物 | 提供二周目动力 |

市场做法:

- Hades 等 Roguelite 通过失败后的永久升级让玩家持续尝试。
- 放置游戏和赛季制游戏常用“重置换永久收益”延长生命周期。

## 7. 当前项目优化清单

### 7.1 P0:必须优先做

#### P0-1. 持久化存储

当前问题:

- `store.py` 是内存 dict。进程重启角色、背包、战斗、修行录都会丢。

为什么优化:

- 长期留存的基础是“玩家相信自己的投入会留下来”。
- 没有持久化,任何日常、图鉴、成就、修行录都不成立。

市场做法:

- 所有长线游戏都会把角色、背包、任务、图鉴、成就持久化。哪怕是单机 Roguelite,也会保存 meta progression。

建议:

- 第一阶段用 SQLite。
- 表:`characters`,`inventories`,`equipped`,`battle_logs`,`journal_entries`,`beastiary`,`daily_state`,`quest_state`。
- 战斗中间态可继续内存,但战斗结束必须写日志。

#### P0-2. 今日修行令

当前问题:

- 主城有入口,但没有“今天该干什么”。

为什么优化:

- 日常目标是 D1/D7 留存最关键的骨架。
- 玩家每天打开游戏时要立刻知道“做什么最值”。

市场做法:

- 星穹铁道 Daily Training 用简单任务给每日稳定奖励。
- 原神/星铁用日常和体力消耗组合形成短会话。

建议:

- Home 顶部新增今日修行卡。
- 每日 5 个目标,完成任意 4 个即可拿满奖励。
- 目标覆盖不同玩法:战斗、打坐、奇遇、赠礼、阅读修行录。
- 每日奖励包含:经验、灵气尘、宗门贡献、山海图点数。

#### P0-3. 材料用途与追踪

当前问题:

- 掉落物很多,但玩家不知道为什么要刷。

为什么优化:

- 刷怪游戏的核心不是“掉东西”,而是“我缺这个东西,所以去刷那个怪”。

市场做法:

- RPG/ARPG/手游普遍提供材料来源、合成用途、目标追踪。

建议:

- Inventory 中每个材料显示“用于:丹方/器方/赠礼/任务”。
- Items 全集增加“追踪此物”。
- ExploreMap 高亮可掉落被追踪材料的怪。

#### P0-4. 山海经图鉴第一版

当前问题:

- 113 怪物内容很丰富,但玩家没有收集进度。

为什么优化:

- 图鉴是最稳定的收集驱动,也能让怪物 lore 不浪费。

市场做法:

- 宝可梦、怪物猎人、各种卡牌/放置游戏都通过图鉴强化收集。

建议:

- 每个怪物记录:遭遇、击败、赠礼成功、掉落已获得、章节已读。
- 每族有总进度:例如狐妖族 3/10。
- 达成族群进度给永久小加成或称号。

#### P0-5. 新手 7 日目标

当前问题:

- 玩家过了第一战后可能不知道长期目标。

为什么优化:

- D1 到 D7 是决定产品是否活下来的关键窗口。

市场做法:

- 手游常用 7 日任务引导玩家接触核心系统,但要避免太重。

建议:

```text
第 1 日:接入灵脉、拜入宗门、首战胜利
第 2 日:完成今日修行令、获得 3 个材料
第 3 日:炼制第一颗丹药、完成一次赠礼
第 4 日:解锁山海经 1 个族群节点
第 5 日:挑战第一个精英怪
第 6 日:完成一条奇遇链
第 7 日:挑战宗门叛徒 Boss 或获得首个法宝
```

奖励以功能解锁和称号为主,不要只堆数值。

### 7.2 P1:一月内形成网状深度

#### P1-1. 山海图节点地图

为什么:

- 把随机探索变成长期拓图。
- 为 Boss、材料、族群、奇遇提供统一入口。

建议:

- 初版 12 个族群区域 + 4 个故事线区域。
- 节点有首通、星级、掉落、危险度。
- 节点可投入“山海图点数”提升怪物等级和掉落。

#### P1-2. 宗门周课

为什么:

- 日常解决短留存,周常解决中期目标。

建议:

- 每周三条路线:猎妖、炼器、问道。
- 完成两条即可拿满基础奖励,三条给额外称号/外观。
- 周课绑定门派偏好,沧澜偏 Boss/叙事,天机偏机关/效率。

#### P1-3. 炼丹炼器

为什么:

- 材料沉淀需要消耗出口。
- 合成目标能自然带动刷怪。

建议:

- 先做 20 个配方。
- 低级丹药解决回血/回灵。
- 中级装备提供构筑差异。
- 高级灵宝接 Boss 和故事线。

#### P1-4. Boss 章节进度

为什么:

- 21 Boss 和 4 故事线是项目内容资产,不能只做百科。

建议:

- Bosses 页面改成故事线地图。
- 每个 Boss 有状态:未知、已闻名、已解锁、可挑战、已首杀。
- 击败低阶 Boss 解锁高阶 Boss 线索。
- 首杀保存 Opus 章节。

#### P1-5. 修行录

为什么:

- LLM 生成文本是本游戏最有传播性的资产。

建议:

- 每场 Boss 首杀、境界突破、奇遇链结局自动入册。
- 支持收藏、标题、标签、导出。
- Home 显示最近修行录,形成“我这个角色有历史”的感觉。

### 7.3 P2:两到三个月形成长期生态

#### P2-1. 天机幻境 Roguelite

玩法:

- 进入 5 层秘境。
- 每层战斗后选一个祝福。
- 祝福影响卡牌、灵气、叙事、掉落。
- 失败也给道基点。

为什么:

- 重复可玩性强。
- LLM 可为每次秘境生成主题和结尾。

市场做法:

- 星穹铁道模拟宇宙、Hades 都证明“局内随机祝福 + 局外成长”极强。

#### P2-2. 洞府闭关/离线收益

玩法:

- 离线期间获得灵气尘、草药、少量修为。
- 收益上限 8-12 小时。
- 可派遣“笔魂”或法宝提升收益。

为什么:

- 面向上班族,降低必须在线刷的压力。

市场做法:

- AFK Journey 等放置游戏的核心就是离线收益 + 回来分配资源。

#### P2-3. 宗门纪行/战令

玩法:

- 免费路线:资源、称号、道基点。
- 高级路线可先不做商业化,只作为内容结构。
- 任务来自日常、周课、Boss、图鉴。

为什么:

- 把分散玩法统一成赛季进度。

市场做法:

- Diablo IV Season Ranks / Battle Pass、各类手游战令都是成熟结构。

#### P2-4. 异步宗门榜

玩法:

- 按宗门统计本周击败 Boss、山海图进度、修行录点赞。
- 不必实时多人,先做单机/本地榜也行。

为什么:

- 宗门身份需要外部比较。

#### P2-5. AI 副本生成器

玩法:

- 玩家输入“我想去一个雷劫主题秘境”。
- Opus 生成 5 层副本结构,系统约束怪物/奖励合法。
- 生成结果进入一次性秘境。

为什么:

- 这是灵枢区别于传统修仙游戏的杀手锏。
- 但必须放在 P2,因为前置的奖励/地图/存档系统要先稳。

## 8. 经济系统建议

### 8.1 资源分层

| 资源 | 来源 | 用途 | 设计目的 |
|---|---|---|---|
| 灵气 | 战斗奖励、打坐、丹药 | 出招、恢复 | 即时战斗资源 |
| 修为 | 战斗、日课、奇遇 | 升级/境界 | 角色成长 |
| 灵气尘/材料 | 掉落、离线 | 炼丹炼器 | 刷怪目标 |
| 宗门贡献 | 日课、周课、Boss | 声望商店 | 宗门身份 |
| 山海图点数 | 首通、周课 | 节点强化 | 地图长期目标 |
| 道基 | 首杀、突破、轮回 | 永久成长 | 长期/转世目标 |
| Token 预算 | 玩家配置 | 控制 LLM 调用 | BYOK 安全感 |

### 8.2 掉落保底

当前 `_roll_drops` 是纯随机。建议加“坏运气保护”:

- 连续 5 次无稀有掉落,下一次稀有率提升。
- 图鉴未解锁的首件材料概率提升。
- 追踪材料时,相关掉落概率 +20%,但每日有上限。

为什么:

- 纯随机会带来挫败。
- 保底不一定要明说全部公式,但要让玩家感觉“刷了就会前进”。

### 8.3 成本透明

BYOK 游戏必须避免玩家害怕:

- 主城显示今日估算 Token 消耗。
- 战斗 HUD 显示本次叙事模型和大概 tokens。
- 提供预算阈值:超过后自动切 speed/模板。
- 高阶模型调用前给“本次为天命/首杀,建议保留演义叙事”。

## 9. 商业化与增长建议

当前最适合的增长不是强付费,而是分享传播。

### 9.1 分享点

| 分享内容 | 形式 |
|---|---|
| Boss 首杀章节 | 长图/Markdown |
| 境界突破 | 角色卡 |
| 天命暴击 | 战报短卡 |
| 山海经收集 | 族群图鉴进度 |
| 宗门周榜 | 榜单截图 |

### 9.2 未来可商业化但不伤核心的方向

- 视觉主题包:宗门皮肤、背景、卡框。
- 叙事风格包:冷峻剑修、热血少年、古文风、轻小说风。

不建议早期做:

- 抽卡式付费战力。
- 纯概率付费洗词条。
- 不透明 token 消耗。

## 12. 最重要的三条产品判断

### 12.1 先把“玩下去的理由”做出来,再继续加内容

项目已经有 113 怪物、21 Boss、55 物品,数量够 MVP 用。现在最缺的是内容之间的连接:

```text
怪物 → 材料 → 配方 → 法宝 → Boss → 故事线 → 山海图 → 周课
```

### 12.2 把 API 成本包装成“修仙资源管理”

BYOK 是门槛,也是差异点。玩家不是单纯在花钱调用模型,而是在调动灵脉。必须让他感到:

- 我知道花在哪里。
- 我能选择省钱模式。
- 关键时刻花高级模型值得。
- 我的高光章节值得保存。

### 12.3 灵枢的核心成瘾点应该是“世界回应我”

传统修仙游戏靠数值和挂机。灵枢应该靠:

- 怪物记得我杀过它同族。
- 宗门因为我的选择改变态度。
- Boss 首杀成为我自己的章节。
- 奇遇链引用我的旧选择。
- 转世后前世痕迹仍然存在。

这才是 LLM 原生游戏最应该打的点。

## 13. 一句话版本

如果只做一个方向:

> **把现有系统连成“山海图 + 今日修行令 + 材料炼制 + 修行录”的四件套。**

这四件套能同时解决:

- 每天上线做什么。
- 为什么要刷怪。
- 掉落有什么用。
- LLM 生成内容怎么沉淀。
- 玩家为什么愿意下次回来。

先有这张网,后续 Boss、赛季、AI 副本、社交榜才会自然长出来。

# 《灵枢笔录》完整游戏玩法总案与逻辑 Review

> 文档目的:把前面分散的修为卡点、门派副本、玩家身世、固定羁绊、打怪动机、四宗统一目标整合成一套可落地的完整玩法。  
> 策划视角:成熟修仙 RPG / 修仙版骑砍,重点解决“为什么要打怪、为什么要升级、为什么要继续玩、为什么要记住这些人”。  
> 当前边界:不设计无 Key 试玩;用户以 BYOK 方式上传自己的 API Key。由于玩家自付模型成本,所有玩法必须兼顾爽感、节奏与消耗透明。

---

## 1. 核心结论

《灵枢笔录》不应该只做“AI 生成战报 + 修仙皮肤”的新鲜体验。新鲜感一过,玩家会发现:

1. 打怪只是刷 token,没有明确目标。
2. 社交人物太多,名字记不住。
3. 升级太快,没有破境卡点。
4. 数值膨胀与境界成长不成体系。
5. 主线目标太弱,无法支撑长期留存。

新的玩法定位应调整为:

**玩家是白石渡裂简夜的幸存者,拜入五宗之一,在修行、打怪、破案、羁绊与门派战争中收集三枚外宗盟印,最终统一本局四大门派。**

简单说,这是:

**修仙版《骑马与砍杀》的门派统一目标 + 修仙小说的破境卡点 + 简单上瘾游戏的短循环反馈。**

## 2. 设计边界与原则

### 2.1 保留的产品前提

| 前提 | 设计结论 |
|---|---|
| 不做无 Key 试玩 | 新用户进入后直接走“选门派 - 填 Key - 验证模型 - 创角”流程 |
| 用户上传自己的 API Key | 所有高消耗叙事节点必须可预估、可跳过、可摘要 |
| 当前已有 5 大门派资产 | 不删除五宗,但单局采用“四宗统一制” |
| 玩家不愿记复杂人名 | 每个阶段只推 3 个关键人物,其余人物留在世界模拟中做背景 |
| 玩家升级过快 | 修为拆分为“真实 token 记录”和“有效修为经验” |
| 玩家缺少打怪欲望 | 怪物掉落必须绑定主线线索、破境材料、羁绊推进与副本门票 |

### 2.2 四条体验原则

1. **短循环必须爽**  
   每 3-5 分钟给一次明确收获:修为、材料、线索、羁绊变化、装备、章节。

2. **中循环必须有卡点**  
   Lv.10、Lv.25、Lv.45、Lv.70 是主线破境门槛,只靠刷怪不能无脑连升。

3. **长循环必须有统一目标**  
   玩家不是漫无目的修仙,而是不断收服外宗,最终召开“四宗归盟台”。

4. **人物必须少而深**  
   不再把 200 个弟子都推给玩家。玩家每阶段只需要记住 1 名核心同伴、1 名暧昧/竞争对象、1 名对立或师长。

## 3. 玩家身份与世界观

### 3.1 玩家统一身世

玩家不是普通散修,而是“白石渡裂简夜”的幸存者。

十年前,白石渡有一页空白本命书从天裂中坠下。五宗同时派人抢夺,渡口村镇一夜覆灭。玩家在废墟中醒来,体内没有完整灵根,却能用文字、战斗、见闻与模型消耗生成“本命书”。

五宗对玩家的态度不同:

| 门派 | 对玩家的解释 | 真实目的 |
|---|---|---|
| 沧澜剑派 | 你是剑冢旧盟的后人 | 想用玩家证明旧盟被背叛 |
| 天机阁 | 你是天机推演中的缺页 | 想补全机关天书 |
| 玄机宗 | 你是因果算法的异常点 | 想验证破局之道 |
| 青冥派 | 你是书火案遗孤 | 想洗清藏书楼旧案 |
| 月隐宫 | 你是被抹名的旧人 | 想取回失落名册 |

这个身世解决三个问题:

1. 玩家和所有门派天然有关联。
2. 玩家有合理理由卷入不同副本。
3. 玩家最终统一四宗不是凭空称王,而是在重建十年前被撕毁的旧盟。

### 3.2 本局四宗制

当前项目有 5 大门派,但用户目标是“统一四大门派”。为兼容资产,采用:

**玩家本门 + 三个外宗 = 本局四宗主线。剩下一个门派作为隐藏宗、观察宗或赛季扩展。**

例:玩家选择沧澜剑派,本局主线为:

| 类型 | 门派 |
|---|---|
| 玩家本门 | 沧澜剑派 |
| 外宗副本 1 | 天机阁 |
| 外宗副本 2 | 青冥派 |
| 外宗副本 3 | 月隐宫 |
| 隐藏宗 | 玄机宗 |

这样不会浪费五宗资产,也不会让单局目标过大。

## 4. 游戏核心循环

### 4.1 30 秒操作循环

```text
选择目标怪物 / 副本节点
→ 战斗
→ 获得有效修为、材料、线索、羁绊推进
→ 血量/灵气/疲劳变化
→ 决定继续打、休息、推进副本、下一轮
```

这一层对应简单上瘾游戏的即时反馈:打完就有东西,但不是只给经验。

### 4.2 20 分钟成长循环

```text
刷怪补修为
→ 收集副本线索
→ 解锁副本节点
→ 击败阶段 Boss
→ 获得盟印 / 破境材料 / 人物关系变化
→ 进入下一个境界段
```

这一层解决玩家“为什么继续打怪”。

### 4.3 多日长期目标

```text
选择门派
→ 完成本门序章
→ 收服外宗一
→ 收服外宗二
→ 收服外宗三
→ 四宗归盟台
→ 统一结局 / 裂盟结局 / 飞升伏笔
```

这一层参考“骑砍式统一目标”:玩家知道自己终点是什么,每次战斗都在推动版图。

## 5. 修为、等级与资源重规划

### 5.1 修为必须拆成两套数

当前实现中,`apply_cultivation_gain()` 会把 `delta_tokens` 同时写入 `cultivation_total`、`token_total` 和 `exp`,导致“打一场怪,token 数就足够升级”。这是升级过快的根因。

新方案:

| 字段 | 含义 | 是否用于升级 |
|---|---|---|
| `token_total` | 玩家真实消耗/生成的 token 总量,用于成本、统计、成就 | 否 |
| `cultivation_total` | 角色累计有效修为,用于长期展示 | 是,但不直接扣 |
| `exp` | 当前等级进度条内的有效修为 | 是 |
| `effective_cultivation_gain` | 本次战斗/章节折算后的有效修为 | 是 |

推荐折算:

```text
普通战斗有效修为 = floor(delta_tokens * 0.10 ~ 0.25) + enemy_level_reward
精英战斗有效修为 = floor(delta_tokens * 0.18 ~ 0.35) + elite_reward
主线 Boss 有效修为 = 固定大额奖励 + 首通奖励
```

这样既保留“燃灵成章”的产品特色,又不会让 token 数直接击穿升级曲线。

### 5.2 境界与等级段

最终主线建议先收束到 Lv.100,不要一开始开放 Lv.200 全体验。

| 境界 | 等级段 | 设计目标 |
|---|---:|---|
| 炼气 | 1-10 | 学会战斗、认识本门、发现白石渡旧案 |
| 筑基 | 11-25 | 第一次外宗副本,建立打怪 - 线索 - 副本的关系 |
| 金丹 | 26-45 | 第二条外宗线,人物羁绊开始出现不可逆选择 |
| 元婴 | 46-70 | 第三条外宗线,门派战争与情感代价爆发 |
| 化神 | 71-100 | 四宗归盟台,统一门派,结算结局 |

当前代码仍保留 46-65、66-85、86-105 等旧分段。实现时必须统一,否则 UI、模型 tier、技能解锁和主线卡点会冲突。

### 5.3 关键修为门槛

| 卡点 | 推荐累计需求 | 破境条件 | 设计体验 |
|---|---:|---|---|
| Lv.10 → Lv.11 筑基 | 约 40,000 | 本门问心试炼 + 筑基材料 | 第一次真正卡点 |
| Lv.25 → Lv.26 金丹 | 约 900,000 | 第一枚外宗盟印 + 金丹材料 | 玩家明白副本比刷怪重要 |
| Lv.45 → Lv.46 元婴 | 约 5,800,000 | 第二枚外宗盟印 + 心魔选择 | 情感线进入主线 |
| Lv.70 → Lv.71 化神 | 约 24,000,000 | 三枚外宗盟印 + 四宗归盟资格 | 进入最终统一线 |

### 5.4 血条、蓝条、疲劳成长

升级不应只是数字 +1,每跨境界应有明显体感。

| 境界 | HP 体感 | 灵气体感 | 疲劳体感 | 战斗变化 |
|---|---|---|---|---|
| 炼气 | 容错低 | 技能少放 | 连续作战有限 | 怪物有压迫感 |
| 筑基 | HP 明显增加 | 可形成基础连招 | 可多打 1-2 场 | 开始刷材料 |
| 金丹 | 生存大幅提升 | 技能循环成型 | 可挑战精英 | 有 build 感 |
| 元婴 | 能扛 Boss 爆发 | 大招进入常规策略 | 长线副本可持续 | 门派战成型 |
| 化神 | 接近统帅级 | 高阶技能主导 | 疲劳管理变成路线规划 | 统一战 |

## 6. 打怪动机重做

### 6.1 怪物不只是经验包

每个怪物必须至少掉落一种和主线相关的东西。

| 掉落类型 | 用途 | 玩家心理 |
|---|---|---|
| 有效修为 | 升级与破境 | 我变强了 |
| 破境材料 | Lv.10/25/45/70 卡点 | 我离下一境更近 |
| 门派线索 | 解锁副本节点 | 我在推进主线 |
| 旧案残证 | 影响副本选择与结局 | 我知道真相更多 |
| 羁绊信物 | 推进固定角色故事线 | 我想知道这个人后来怎样 |
| 装备/丹药材料 | 强化战斗 build | 我下一场会更强 |

### 6.2 掉落与等级段绑定

| 等级段 | 主掉落 | 副掉落 | 目标 |
|---|---|---|---|
| Lv.1-10 | 筑基材料、白石渡残片 | 本门人物信物 | 推进本门序章 |
| Lv.11-25 | 第一外宗线索 | 金丹材料 | 打开第一个外宗副本 |
| Lv.26-45 | 第二外宗旧案残证 | 元婴心魔材料 | 打开第二个外宗副本 |
| Lv.46-70 | 第三外宗战争情报 | 化神归盟凭证 | 打开第三个外宗副本 |
| Lv.71-100 | 归盟令、旧盟血书 | 结局旗帜 | 推进最终统一 |

### 6.3 战斗结果要服务主线

战斗胜利不再只显示:

```text
获得 300 修为。
```

而应显示:

```text
获得有效修为 420
获得青冥书火残页 1/3
唐灵枢羁绊 +5
四宗归盟线索:藏书楼火势并非从内门烧起
```

这会让玩家觉得每场战斗都在推动剧情。

## 7. 等级地图与一键扫荡系统

### 7.1 设计目的

现有探索地图是“按玩家等级随机刷一批怪”。这个结构适合新鲜感,但不利于长期留存。玩家需要更明确的地图进度:

```text
进入地图
→ 选择等级地图
→ 刷怪收集资源
→ 挑战本图 Boss
→ Boss 首通后解锁一键扫荡
→ 用疲劳额度快速换取该图物资
→ 用物资突破、炼丹、强化、推进副本
```

这套系统解决三个问题:

1. 玩家知道自己现在应该刷哪张图。
2. 每张图有一个明确 Boss 目标。
3. Boss 打过后,低等级重复劳动可以转为扫荡,减少疲劳感。

### 7.2 地图生成规则

玩家第一次进入探索页时,系统不再只生成一张随机地图,而是初始化至少 10 张固定等级地图。

初始化字段:

```ts
map_progress = {
  white_stone_shoal: {
    unlocked: true,
    boss_defeated: false,
    sweep_unlocked: false,
    normal_clears: 0,
    boss_clears: 0,
    sweep_count: 0
  },
  fox_mist_forest: {
    unlocked: true,
    boss_defeated: false,
    sweep_unlocked: false,
    normal_clears: 0,
    boss_clears: 0,
    sweep_count: 0
  }
}
```

地图不应该每次刷新都随机变化。随机只发生在“当前地图内刷哪些怪、怪物站位、精英概率、掉落浮动”。

设计注意:

1. 地图推荐等级不再大面积重叠,每张图有清晰等级边界。
2. 普通怪、精英怪、Boss 都必须落在本图等级范围内。
3. 每张图可以有多个 Boss,但至少一个镇图 Boss 负责解锁扫荡。
4. 地图 Boss 是资源产线 Boss,不是主线最终 Boss。
5. 高等级地图可以先显示轮廓,但未达等级时锁定。
6. 地图名称要直接暗示物资用途,玩家一看就知道去哪刷。

### 7.4 地图解锁节奏

| 玩家阶段 | 开放地图 | 目的 |
|---|---|---|
| Lv.1-10 炼气 | 地图 1-2 | 新手材料、筑基准备 |
| Lv.11-25 筑基 | 地图 2-3 | 第一外宗副本线索、金丹准备 |
| Lv.26-45 金丹 | 地图 4-5 | 第二外宗副本线索、元婴准备 |
| Lv.46-70 元婴 | 地图 6-8 | 第三外宗副本线索、化神准备 |
| Lv.71-100 化神 | 地图 8-10 | 归盟令、终局材料 |

地图解锁条件建议:

```text
基础条件:玩家等级达到地图最低推荐等级 - 2
剧情条件:对应主线 Act 已开启
特殊条件:部分地图需要前置地图 Boss 首通
```

例:

```text
龙渊水府解锁条件:
- 玩家 Lv.64+ 可预览 / Lv.66+ 推荐进入
- 已获得第二枚外宗盟印
- 青冥药谷 Boss 已首通
```

### 7.5 Boss 首通与扫荡解锁

每张地图都有一个地图 Boss。玩家首次击败该 Boss 后,该地图解锁一键扫荡。

首通奖励:

| 奖励 | 说明 |
|---|---|
| 大量有效修为 | 只给一次,强化首通爽感 |
| 地图核心物资 | 让玩家能立刻制作/突破 |
| 地图扫荡权限 | `sweep_unlocked = true` |
| 地图图鉴章 | 进入本命书或地图志 |
| 少量主线线索 | 只作为辅助,不能直接跳过主线副本 |

Boss 首通状态:

```ts
map_progress[map_id].boss_defeated = true
map_progress[map_id].sweep_unlocked = true
map_progress[map_id].boss_clears += 1
```

### 7.6 一键扫荡规则

扫荡只消耗疲劳额度,不进入战斗、不消耗 HP/灵气、不生成长篇 LLM 战报。

当前代码中的疲劳是“行动后疲劳上升”的资源。为了和用户感知统一,UI 文案可写为:

```text
消耗疲劳额度 8
```

系统实际表现:

```text
fatigue += sweep_cost
```

扫荡条件:

| 条件 | 说明 |
|---|---|
| 地图已解锁 | `unlocked = true` |
| Boss 已首通 | `sweep_unlocked = true` |
| 疲劳未满 | `fatigue + sweep_cost <= max_fatigue` |
| 玩家等级未严重低于地图 | 防止低等级靠他人代打后刷高阶资源 |

扫荡次数:

| 按钮 | 效果 |
|---|---|
| 扫荡 1 次 | 消耗一次疲劳,获得一次掉落 |
| 扫荡 5 次 | 连续结算 5 次,疲劳不足则自动停 |
| 扫荡至疲劳满 | 按剩余疲劳自动计算最大次数 |

### 7.7 扫荡奖励边界

扫荡必须爽,但不能替代所有玩法。

扫荡可以产出:

1. 常规材料。
2. 少量有效修为。
3. 低概率地图稀有材料。
4. 已解锁副本所需的普通线索。
5. 少量灵气或货币类资源。

扫荡不能产出:

1. 地图 Boss 首通奖励。
2. 外宗盟印。
3. 主线 Boss 关键选择。
4. 固定羁绊重大事件。
5. 破境心魔结论。

这样可以避免玩家“扫荡替代主线”,同时保留刷资源的爽感。

### 7.8 不同地图的扫荡价值

| 地图 | 推荐等级 | 扫荡有效修为/次 | 普通材料 | 稀有掉率 | 用途 |
|---|---:|---:|---|---:|---|
| 白石渡残滩 | Lv.1-8 | 120 | 1-3 个 | 2% | 筑基草、白石残片 |
| 终南狐林 | Lv.9-16 | 240 | 1-3 个 | 3% | 幻术残页、筑基丹引 |
| 灵雀断崖 | Lv.17-25 | 420 | 2-4 个 | 4% | 雷核碎片、金丹前置 |
| 寒溪蛇窟 | Lv.26-35 | 700 | 2-4 个 | 5% | 金丹药引 |
| 白虎断坡 | Lv.36-45 | 1,100 | 2-5 个 | 6% | 金丹铸体、兽王血 |
| 青冥药谷 | Lv.46-55 | 1,600 | 3-5 个 | 7% | 元婴心魔引 |
| 鬼灯旧市 | Lv.56-65 | 2,300 | 3-6 个 | 8% | 书火残证 |
| 龙渊水府 | Lv.66-75 | 3,200 | 3-6 个 | 10% | 化神归盟材 |

推荐收益关系:

```text
手动精英战收益 = 扫荡同图 2.0-2.5 次
手动普通战收益 = 扫荡同图 1.3-1.5 次
镇图 Boss 首通收益 = 扫荡同图 10 次
稀有 Boss 首通收益 = 扫荡同图 6 次
```

### 7.9 与主线副本的关系

地图系统是资源线,副本系统是剧情线。两者关系如下:

| 系统 | 作用 |
|---|---|
| 等级地图 | 刷修为、刷材料、刷普通线索 |
| 地图 Boss | 解锁扫荡、给阶段爽点 |
| 外宗副本 | 推进主线、获得盟印、决定人物命运 |
| 破境试炼 | 检查修为、材料、盟印、羁绊 |

例:

```text
玩家卡在 Lv.25 → Lv.26 金丹
尚缺:
- 金丹药引 2
- 第一外宗盟印

玩家可以:
- 去寒溪蛇窟扫荡金丹药引
- 去第一外宗副本挑战主线 Boss 获得盟印
```

这会让玩家很清楚:扫荡能解决资源,但真正破境仍要推进主线。

| 风险 | 说明 | 处理方式 |
|---|---|---|
| 扫荡让玩家不打怪 | 扫荡太强会替代战斗 | 扫荡收益低于手动精英战,且不产关键剧情物 |

| 主线被资源线稀释 | 玩家只扫荡不做副本 | 破境必须要盟印和心魔结论 |

### 8.2 副本标准结构

每个外宗副本固定 7 个节点,降低开发和玩家理解成本。

| 节点 | 类型 | 内容 |
|---|---|---|
| 1 | 入口事件 | 外宗请求、密信、追杀、旧案证人 |
| 2 | 小怪战 | 收集第一组线索 |
| 3 | 人物抉择 | 同伴、暧昧对象、对立者三方出现 |
| 4 | 精英战 | 掉落关键残证 |
| 5 | 真相反转 | 玩家发现本门也有责任 |
| 6 | Boss 战 | 击败外宗核心危机 |
| 7 | 盟印结算 | 收服、和盟、逼盟、裂盟四种结果 |

### 8.3 盟印不是单纯通关章

外宗 Boss 死亡不等于获得盟印。盟印结果取决于:

1. 副本主线完成度。
2. 关键人物羁绊。
3. 玩家是否隐瞒真相。
4. 是否保住外宗核心利益。
5. 是否牺牲某个角色。

推荐结算:

| 结果 | 条件 | 效果 |
|---|---|---|
| 真盟 | 真相公开 + 核心人物存活 + 好感达标 | 最强盟印,最终战援军 |
| 和盟 | 真相部分公开 + Boss 击败 | 标准盟印 |
| 逼盟 | 武力压服 + 好感不足 | 获得盟印,但最终战有反噬 |
| 裂盟 | Boss 击败但关键人物死亡/背叛 | 可进入主线,但结局变差 |

## 10. 固定羁绊系统

### 10.1 为什么要固定羁绊

当前世界模拟有 200 名弟子和大量关系,作为背景很丰富,但对普通玩家太重。玩家需要的是:

1. 少数能反复出现的人。
2. 关系能持续变化。
3. 情感选择能影响副本和结局。

因此玩家主线只保留 12 名固定羁绊角色,每局前期只激活 3-4 名。

### 10.2 角色功能分层

| 角色类型 | 功能 | 玩家记忆点 |
|---|---|---|
| 本门同伴 | 陪玩家从弱到强 | 友情和信任 |
| 外宗暧昧/竞争者 | 制造情感牵引 | 爱情、误会、三角关系 |
| 师长/操盘者 | 推动主线真相 | 师徒、利用、背叛 |
| 对立者 | 让副本有冲突 | 仇怨、立场、救赎 |

### 10.3 下一轮触发方式

点击“下一轮”时,除了现有 NPC 世界事件,还应偶尔触发玩家羁绊事件。

触发条件:

| 条件 | 事件倾向 |
|---|---|
| 玩家刚升级 | 同伴祝贺、师长提醒、竞争者挑衅 |
| 玩家卡在破境 | 师徒问心、暧昧对象送材料、对立者设局 |
| 玩家完成副本节点 | 外宗角色私信、隐藏真相、三角冲突 |
| 玩家连续打怪 | 同伴受伤、怪物旧案牵出人物 |
| 玩家疲劳过高 | 关心、争吵、误会、治疗 |

这样“下一轮”不再只是看世界自己动,而是玩家会期待“这次会不会有人来找我”。


### 11.3 战斗掉落规则

战斗结算新增一个故事掉落函数:

```python
def roll_story_drops(char, enemy, battle_result):
    progress = char.get("main_story_progress", {})
    act = progress.get("current_act", 1)
    active_dungeon = progress.get("active_dungeon")
    drops = []

    if battle_result != "victory":
        return drops

    drops += roll_breakthrough_materials(char["level"], enemy.level)
    drops += roll_active_dungeon_clues(active_dungeon, enemy)
    drops += roll_bond_tokens(progress, enemy)
    return drops
```

掉落优先级:

1. 玩家卡破境时,优先掉破境材料。
2. 玩家有 active dungeon 时,优先掉对应宗门线索。
3. 玩家近期触发过羁绊事件时,小概率掉对应信物。
4. Boss 必掉关键残证或盟印推进物。

### 11.4 有效修为结算

`apply_cultivation_gain()` 不再把 `delta_tokens` 直接加入 `exp`。

推荐伪代码:

```python
raw_tokens = delta_tokens
effective_gain = calculate_effective_cultivation(raw_tokens, source, char, task)

char["token_total"] += raw_tokens
char["cultivation_total"] += effective_gain
char["daily_token_used"] += raw_tokens
char["monthly_token_used"] += raw_tokens
char["exp"] += effective_gain
```

其中 `calculate_effective_cultivation()` 根据来源分档:

| 来源 | 有效修为倍率 |
|---|---:|
| 普通战斗章节 | 10%-18% |
| 精英战斗章节 | 18%-28% |
| 主线 Boss 章节 | 固定奖励 + 25%-35% |
| 破境章节 | 不给大量经验,给境界奖励 |
| 下一轮羁绊章节 | 少量修为 + 羁绊 |

### 11.5 破境检查

新增:

```python
def can_breakthrough(char):
    level = char["level"]
    progress = char.get("main_story_progress", {})

    if level == 10:
        return has_exp(char) and progress["breakthrough_flags"]["foundation_trial_done"] and has_item("pill_foundation")
    if level == 25:
        return has_exp(char) and has_first_alliance_token(progress) and has_item("golden_core_material")
    if level == 45:
        return has_exp(char) and has_second_alliance_token(progress) and has_flag("heart_demon_resolved")
    if level == 70:
        return has_exp(char) and has_three_alliance_tokens(progress) and has_item("old_alliance_blood_oath")
    return has_exp(char)
```


## 12. 完整玩家体验流程

### 12.1 新手前 30 分钟

1. 选择门派。
2. 上传 API Key 并验证模型。
3. 看到自己的白石渡身世。
4. 完成 2-3 场轻量战斗。
5. 触发第一个固定同伴事件。
6. 发现本门也隐瞒了白石渡真相。
7. 到达 Lv.10,出现第一次破境卡点。
8. 完成本门问心试炼,筑基成功。

目标:让玩家知道这不是纯刷怪游戏,而是“我在查自己的身世”。

### 12.2 第一个留存钩子

筑基后立刻给玩家一个外宗选择预告,但只开放第一条:

```text
天机阁来信:十年前的机关锁芯,在你醒来的白石渡水下再次转动。
```

玩家此时已经有:

1. 明确目标:去天机阁副本。
2. 明确缺口:需要 3 个机关锁残片。
3. 明确人物:墨听潮/陈墨痕/顾千澜其中一人将陪同。

### 12.3 中期持续动力

每个外宗副本必须包含:

1. 一个能理解的门派矛盾。
2. 一个玩家个人身世线索。
3. 一个固定人物情感推进。
4. 一个会影响结局的选择。
5. 一个强 Boss 和一枚盟印。

玩家继续玩的理由不是“等级更高”,而是:

```text
我想知道十年前谁害了我。
我想知道这个人会不会背叛我。
我想把这个宗门拉到我这边。
我想看看最终四宗归盟会怎样。
```

### 12.4 最终目标

化神阶段开启“四宗归盟台”。

结局不只看是否打赢最终 Boss,还看三枚盟印质量:

| 盟印组合 | 结局 |
|---|---|
| 3 真盟 | 四宗归心,玩家成为新盟主 |
| 2 真盟 + 1 和盟 | 统一成功,但隐藏宗留下伏笔 |
| 含逼盟 | 统一成功,但战后反噬 |
| 含裂盟 | 勉强统一,关键人物离开或死亡 |
| 未集齐 | 无法开启归盟台 |

## 13. 数值与战斗体验

### 13.1 数值原则

1. 小等级增长要稳定,跨境界增长要明显。
2. 怪物难度要围绕玩家等级段生成,不要重复乘境界倍率。
3. Boss 难度来自机制、技能和阶段,不是单纯血量暴涨。
4. 疲劳不是惩罚玩家,而是引导玩家轮换玩法。

### 13.2 当前风险

现有数值文档已指出:

1. 玩家 `derive_stats()` 使用 `realm_multiplier(level)`。
2. 怪物进入战斗也可能乘等级倍率。
3. NPC 如果提前算过最终属性,再进战斗乘倍率,会双重膨胀。

因此后续实现副本 Boss 时,必须给 Boss 增加 `pre_scaled=True` 或统一用“基础值 + 战斗引擎单次缩放”的规范。

### 13.3 推荐战斗时长

| 战斗类型 | 回合数 | 目的 |
|---|---:|---|
| 普通怪 | 2-5 回合 | 快速反馈 |
| 精英怪 | 5-8 回合 | 检查 build 和资源 |
| 副本 Boss | 8-14 回合 | 叙事高潮 |
| 归盟台最终战 | 12-20 回合 | 多阶段压迫 |

## 14. 社交信息减负

### 14.1 信息分层

| 层级 | 展示方式 | 玩家需要记吗 |
|---|---|---|
| 主线羁绊 | 首页、下一轮、破境、副本强展示 | 需要 |
| 外宗关键人物 | 当前副本页展示 | 需要,但阶段性 |
| 200 弟子世界模拟 | 世界动态、人物志、传闻 | 不强迫 |
| 普通 NPC 关系 | 后台生成 | 不需要 |

### 14.2 一屏只给 3 个名字

任何主线节点最多同时突出:

1. 一个同伴。
2. 一个对立者。
3. 一个情感/师徒角色。

超过 3 个名字就会稀释记忆。

### 14.3 人名必须绑定标签

不要只显示:

```text
唐灵枢 好感 +5
```

应显示:

```text
唐灵枢 · 青冥藏书楼幸存者
好感 +5
她交给你半页烧焦的经文。
```

标签比名字更容易被普通玩家记住。

## 15. BYOK 成本体验

因为玩家要上传自己的 API Key,玩法必须避免“我在玩游戏,但不知道烧了多少钱”的焦虑。

### 15.1 成本透明

每个高消耗动作前显示:

| 信息 | 示例 |
|---|---|
| 预计 token | 约 800-1200 tokens |
| 叙事价值 | 生成一章主线战报 |
| 玩法奖励 | 副本线索 + Boss 奖励 |
| 可选项 | 简略结算 / 完整成章 |

### 15.2 分层生成

| 内容 | 是否必须 LLM |
|---|---|
| 普通怪数值结算 | 否 |
| 普通怪短战报 | 可选 |
| 主线 Boss 战报 | 是 |
| 破境章节 | 是 |
| 下一轮普通世界事件 | 否 |
| 下一轮玩家羁绊关键事件 | 可选 LLM |

这样既不违背 BYOK 前提,又能把成本花在玩家最在意的节点上。

## 16. 逻辑 Review

### 16.1 已解决的逻辑问题

| 问题 | 原风险 | 总案修正 |
|---|---|---|
| 五宗资产 vs 四宗统一 | 世界观冲突 | 单局四宗制 + 隐藏宗 |
| 用户记不住人名 | 200 人同时出现 | 每阶段只强推 3 个关键角色 |
| 打怪无欲望 | 只有经验和 token | 怪物掉落绑定主线线索、材料、羁绊 |
| 升级太快 | token 直接转 exp | token_total 与 effective exp 分离 |
| 主线弱 | 没有长期目标 | 三枚盟印 + 四宗归盟台 |
| 社交冗余 | 关系网复杂但不服务玩家 | 固定羁绊进入副本和破境 |
| 刷怪重复疲劳 | 低等级资源需要反复打 | 10 张地图 + Boss 首通扫荡 |

### 16.2 仍需实现时注意的 P0 冲突

| 冲突点 | 当前位置 | 风险 | 必须处理 |
|---|---|---|---|
| 升级公式仍是 `(level + 1) * 100` | `backend/app/attributes.py`, `frontend/src/components/StatusBar.vue` | 新卡点完全失效 | 改为统一曲线并由后端返回需求 |
| `delta_tokens` 直接加 `exp` | `backend/app/cultivation.py` | 一场战斗可能连升多级 | 拆 raw token 和有效修为 |
| 境界分段仍是旧 9 境到 Lv.200 | `backend/app/sects.py` | 与 Lv.100 主线卡点冲突 | 先统一主线 1-100,高阶作为后续赛季 |
| 世界模拟没有玩家主线字段 | `frontend/src/utils/worldSimulation.js` | 下一轮仍只像旁观世界 | 加入 player_bond_events 与 main_story_progress |
| 副本掉落未接战斗结算 | `backend/app/battle.py` / reward 相关逻辑 | 打怪仍不服务主线 | 增加 story drop roll |
| 破境自动升级未检查剧情条件 | `check_level_up()` | 玩家可越过主线 | 卡点等级必须停住 |
| 探索地图只有随机刷怪 | `frontend/src/views/ExploreMap.vue`, `/api/explore/spawn` | 无地图进度、无扫荡、资源目标不清晰 | 新增固定地图配置、map_progress、sweep 接口 |

### 16.3 设计层面的风险

| 风险 | 说明 | 缓解 |
|---|---|---|
| 卡点过硬导致挫败 | 玩家可能觉得被强制刷材料 | 每个缺口给明确来源和一键追踪 |
| 剧情过重影响爽感 | 文字太多会拖慢节奏 | 普通战斗短反馈,主线节点才长叙事 |
| BYOK 成本吓退用户 | 玩家担心每一步都烧钱 | 默认普通结算低消耗,关键节点提示成本 |
| 统一目标太远 | 新手看不到终点 | 首页显示“当前盟印 0/3、归盟进度” |
| 情感线争议 | 爱情/三角恋处理不好会尴尬 | 只写克制的修仙式羁绊,以选择和牺牲推动 |
| 扫荡过强 | 玩家不再手动打怪 | 扫荡只给常规资源,不产盟印和关键剧情 |

### 16.4 最终判断

从逻辑上看,这套玩法可以成立,但前提是必须把“数值卡点”和“主线副本”绑定实现。  
如果只改剧情文案,不改升级公式和掉落结构,玩家仍然会快速升级、快速失去目标。  
如果只改数值,不改主线与羁绊,玩家会觉得变成纯刷子。  

因此优先级必须是:

```text
P0: 修为曲线 + 破境检查 + 主线进度字段
P1: 战斗掉落线索 + 副本节点解锁
P2: 下一轮羁绊事件 + 四宗归盟结局
```

## 17. 开发拆分建议

### 17.1 第一阶段:让升级卡住

目标:解决“升级太快”。

任务:

1. 新增统一修为曲线。
2. `apply_cultivation_gain()` 改为 raw token 与 effective cultivation 分离。
3. `check_level_up()` 在 Lv.10/25/45/70 检查破境条件。
4. 前端状态栏读取真实 `next_level_need`。
5. UI 显示破境缺口。

### 17.2 第二阶段:让打怪有目标

目标:解决“为什么打怪”。

任务:

1. 新增 story drop table。
2. 普通怪掉材料与线索。
3. 精英怪掉旧案残证。
4. Boss 掉副本关键物。
5. 战斗结算页展示“主线推进”。

### 17.3 第三阶段:让地图可扫荡

目标:解决“重复刷低等级材料太累”。

任务:

1. 新增 10 张固定地图配置。
2. 玩家首次进探索页时初始化 `map_progress`。
3. `/api/explore/spawn` 支持 `map_id`。
4. 地图 Boss 首通后写入 `sweep_unlocked`。
5. 新增地图扫荡接口,只消耗疲劳额度并结算常规物资。

### 17.4 第四阶段:让副本成为主线

目标:解决“剧情不吸引人”。

任务:

1. 创建路线配置。
2. 每个门派生成 3 条外宗副本。
3. 副本节点按线索解锁。
4. Boss 结算盟印质量。
5. 盟印影响最终归盟台。

### 17.5 第五阶段:让下一轮有期待

目标:解决“社交信息冗余但不动人”。

任务:

1. 在下一轮中加入玩家羁绊事件。
2. 每次只推 1 条玩家相关事件。
3. 羁绊事件影响副本选择和破境心魔。
4. 重要羁绊进入本命书。

## 19. 最终玩法一句话

玩家上传自己的 API Key,选择一个门派成为白石渡幸存者,通过 10 张等级地图打怪获得有效修为、破境材料和门派线索,击败地图 Boss 后解锁一键扫荡来快速获取常规物资,在 Lv.10/25/45/70 的境界卡点挑战主线副本,与少数固定角色产生友情、爱情、师徒、背叛和三角纠葛,逐步收服三个外宗,最终在四宗归盟台统一本局四大门派。

这套玩法的核心吸引力不是“AI 会写东西”,而是:

**每一场战斗都让玩家离破境、真相、某个人和统一四宗更近一步。**

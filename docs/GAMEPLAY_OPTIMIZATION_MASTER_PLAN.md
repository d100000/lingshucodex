# 灵枢笔录 · 爽感玩法优化总案

> 日期: 2026-05-26  
> 视角: 成熟商业游戏策划评估  
> 范围: 在现有《灵枢笔录》框架上增强爽感、留存和玩法深度  
> 硬约束: 不做免 Key 试玩。用户必须提交并验证自己的 API Key 后才能进入正式游戏。

---

## 0. 总结结论

《灵枢笔录》当前最强的资产不是怪物数量,也不是单场战斗文案,而是:

```text
API Key = 灵脉
Token = 燃灵墨 / 灵气火候
模型档位 = 境界
战斗与章节 = 玩家本命书的一部分
```

这套设定足够独特,但目前的爽感还没有完全释放。普通玩家进入后容易遇到三类问题:

| 问题 | 玩家体感 | 策划判断 |
|---|---|---|
| 接入门槛高 | 我到底要填什么,会花多少钱 | BYOK 是核心,但必须做成可信任的入门仪式 |
| 高频战斗等待 | 我点了招式,但爽点被 LLM 等待稀释 | 数值反馈必须即时,文学推演要分层 |
| 成长目标发散 | 我赢了,但下一步该干什么 | 战斗、掉落、炼制、技能、图鉴、本命书要形成闭环 |

因此优化方向不是继续堆更多怪物,而是建立五个爽感支柱:

```text
1. 必须 Key,但接入过程可信、透明、短。
2. 小怪战斗要快,精英/Boss/突破再重叙事。
3. 战斗要有敌人意图、连招、斩杀、战后高光。
4. 掉落必须立刻通向装备、炼制、技能或图鉴目标。
5. 每局行为都沉淀到本命书、山海经、Boss 线索或宗门关系。
```

推荐实施顺序:

```text
P0: Key 接入信任层 + 极速小怪战 + 敌人意图 + 连招/斩杀 + 战后高光
P1: 装备可穿戴 + 技能构筑 + 破境仪式 + 推荐目标
P2: 秘境 Roguelite + 山海经研究 + Boss 线索 + 战报分享
P3: 宗门周课 + 赛季纪元 + 社交展示
```

## 1. 核心产品立场

### 1.1 不做试玩,但不能让 Key 成为劝退点

本方案遵守硬约束:

```text
未提交并验证 API Key 的用户,不能进入主城、探索、战斗、秘境或本命书生成。
```

但从成熟产品角度看,不做试玩不代表可以把接入体验做得复杂。相反,因为用户要交出自己的 Key,游戏必须更早建立信任。

必须补三件事:

| 功能 | 目的 | 当前框架契合 |
|---|---|---|
| Key 成本契约 | 告诉玩家每类行为大概消耗多少 token | 接 `cultivation` / `battle` usage |
| Key 安全说明 | 明确是否存储、如何删除、是否加密 | 接 `ByokSettings` / `character/me/byok` |
| 探测结果解释 | 不只是显示可选门派,还要告诉用户为什么不可选 | 接 `health_check.py` probe |

重要风险:

当前 `Onboarding.vue` 文案写着“平台不存储到数据库”,但后端 `choose_sect` 会把 `api_key` 写入角色存档,只是返回前隐藏。这个不一致必须修正。

成熟策划建议:

```text
要么改成“后端加密保存,仅用于你的战斗与成章,可随时删除/更换”;
要么改成真正不落库,由客户端安全保存并每次请求带入。
```

如果继续服务端保存,建议优先加:

- API Key 加密存储。
- 一键删除 Key。
- 更换 Key 前后显示 mask。
- 明确 usage 来源: provider 精确值 / 系统估算值。
- 每日/月度预算提醒,默认只提醒不强制限流。

### 1.2 爽感不是快,而是“点一下马上有结果”

LLM 文字是特色,但不是每次点击都要等待 LLM 才能爽。成熟游戏的即时反馈通常由三层组成:

| 层级 | 时间 | 作用 |
|---|---:|---|
| 即时层 | 0-300ms | 数字飞出、血条变化、震动、音效/闪光 |
| 判断层 | 0.5-2s | 连招、斩杀、掉落、升级、任务推进 |
| 叙事层 | 2-15s | 本命书、Boss 剧情、突破章、关键高光 |

当前战斗引擎已经支持“数值即时结算 + LLM 后台推演”,应继续强化这个方向。

### 1.3 小怪负责节奏,Boss 负责记忆点

建议按内容价值分层:

| 内容 | 推荐节奏 | LLM 策略 |
|---|---|---|
| 普通小怪 | 快速击杀,10-25 秒一局 | 默认极速结算,可战后手动成章 |
| 首遇怪物 | 稍重仪式,30-60 秒 | 短叙事 + 图鉴首遇 |
| 精英怪 | 有策略,1-3 分钟 | 回合短叙事 + 掉落目标 |
| Boss | 高仪式,3-8 分钟 | 全量叙事 + 战后大章 |
| 境界突破 | 高仪式,1-3 分钟 | 专属破境章 |

极速战斗不等于试玩,也不等于绕过 Key。它只是对已经验证 Key 的正式玩家提供高频刷怪节奏。

| 暂缓事项 | 原因 |
|---|---|
| 继续堆怪物数量 | 已有 113 种,短期瓶颈是用途和目标感 |

| 免 Key 试玩 | 与当前产品立场冲突 |

## 3. P0 功能:必须先补的爽感底座

### 3.1 Key 接入信任层

#### 功能目标

让用户提交 Key 前明确知道:

```text
我需要什么 Key
这个 Key 能加入哪些门派
每场战斗大概消耗多少
我的 Key 会如何保存和删除
我可以在哪里看使用记录
```

#### 设计方案

在 `/onboarding` 加一个“灵脉契约”区块:

| 内容 | 展示方式 |
|---|---|
| Key 状态 | 未接入 / 已探测 / 已验证 / 需更换 |
| API 地址 | 预设按钮 + 手动输入 |
| 可用模型 | 按门派聚合,显示缺失模型 |
| 预计消耗 | 小怪极速、正传推演、Boss、本命书章节 |
| 安全说明 | 是否服务端保存、是否加密、如何删除 |
| 预算保护 | 今日提醒阈值、单章提醒阈值 |

#### 当前框架落点

- 前端: `frontend/src/views/Onboarding.vue`
- 前端: `frontend/src/components/ByokSettings.vue`
- 后端: `backend/app/health_check.py`
- 后端: `backend/app/main.py` 的 `/api/byok/probe`、`/api/character/me/byok`
- 后端: `backend/app/cultivation.py` usage 统计

#### 成熟策划评估

| 维度 | 评估 |
|---|---|
| 用户爽感 | 中。它不直接爽,但会降低焦虑,让后续爽感能发生 |
| 留存价值 | 高。BYOK 产品最怕首日流失在不信任 |
| 开发成本 | 中 |
| 风险 | 安全文案与实际实现不一致会伤害信任 |
| 优先级 | P0 |

#### 验收指标

- 70% 以上新注册用户能完成 probe。
- Key 验证失败用户能看懂失败原因。
- `/onboarding` 不再出现与后端存储行为冲突的文案。
- `More/ByokSettings` 可明确查看 mask、usage、删除/更换入口。

### 3.2 新手“三步入道”

#### 功能目标

不做试玩,但用户验证 Key 后必须在 3 分钟内完成第一个完整闭环:

```text
第一战 -> 第一份掉落 -> 第一次强化/炼制/技能提升
```

#### 流程设计

```text
Key 验证通过
  -> 选派 / 拜入
  -> 系统自动给“入道三步”
     1. 击败山林小狐或小灵雀
     2. 查看掉落材料用途
     3. 使用丹药 / 精进技能 / 炼制低级物品
  -> 完成后解锁常规主城布局
```

#### 设计要点

- 第一战敌人必须低风险。
- 第一战必须展示“为什么选这个怪”。
- 战后结算必须给下一步按钮,不是只给“回地图”。
- 若掉落不足,给保底材料,保证首次闭环不断。

#### 当前框架落点

- `daily.py` 已有今日修行令,可新增 `novice_path`。
- `Battle.vue` 已支持 `tutorial_hint`。
- `recipes.py` 已有低级配方。
- `Inventory.vue` 可加“可用于”入口。

#### 成熟策划评估

| 维度 | 评估 |
|---|---|
| 用户爽感 | 高。第一局结束后立刻变强 |
| 留存价值 | 高。首个目标闭环是 D1 的基础 |
| 开发成本 | 低-中 |
| 风险 | 引导太重会像教程,需可跳过但不要默认隐藏 |
| 优先级 | P0 |

### 3.3 小怪极速战默认化

#### 功能目标

普通小怪战斗必须短、脆、连续。让玩家在 10 分钟内能完成多次“选怪-出招-掉落-强化”。

#### 模式设计

| 模式 | 使用场景 | LLM 行为 |
|---|---|---|
| 极速战 | 普通小怪、重复刷材料 | 数值即时结算,不等待回合叙事 |
| 正传战 | 首遇、精英、Boss、突破 | 回合叙事 + 战后章节 |
| 手动成章 | 极速战后玩家觉得值得记录 | 消耗玩家 Key,生成本命书片段 |

注意:

```text
极速战仍要求用户已经上传并验证 API Key。
它是节奏优化,不是免 Key 内容。
```

#### 触发规则

- 普通怪默认极速。
- 首次遇见某族或某怪,默认正传。
- 高于玩家 5 级以上,默认正传。
- Boss、秘境 Boss、境界突破必定正传。
- 玩家可在设置里改默认偏好。

#### 当前框架落点

- 后端 `battleApi.start(enemy_id, mode)` 已支持 `speed/drama`。
- `battle.py` 已在 speed 模式跳过章节等待。
- `ExploreMap.vue` 双击开战时增加 mode 选择或默认判断。
- `Battle.vue` 结算页增加“燃灵成章”按钮。

#### 成熟策划评估

| 维度 | 评估 |
|---|---|
| 用户爽感 | 极高。直接提升刷怪节奏 |
| 留存价值 | 高。材料、技能、日课都依赖高频战斗 |
| 开发成本 | 中 |
| 风险 | 若所有战斗都极速,AI 特色会变弱;需让首遇/Boss 保持仪式感 |
| 优先级 | P0 |

### 3.4 敌人意图系统

#### 功能目标

让玩家不是“无脑点最高伤害”,而是在出牌前看见对方下回合趋势。

#### 意图类型

| 意图 | UI 文案 | 玩家应对 |
|---|---|---|
| 攻击 | 下回合将扑击 | 正常输出或护体 |
| 重击 | 正在蓄力 | 防御、打断、斩杀 |
| 闪避 | 准备游走 | 用命中/速度类技能 |
| 守势 | 收缩防御 | 用破防或 buff |
| 回血 | 准备吞灵 | 尽快压血线 |
| 逃离 | 心生退意 | 追击或赠礼 |

#### 后端规则

用轻量 FSM 替代纯随机:

```text
HP > 60%: 攻击 / 守势
HP 30%-60%: 根据族群偏好选择重击 / 闪避 / 毒 / buff
HP < 30%: 回血 / 逃离 / 终极技
玩家连续暴击: 敌人下回合更容易重击
玩家连续防御: 敌人更容易 buff 或蓄力
```

#### 当前框架落点

- `monster_skills.py`: 增加 `preview_intent_for_enemy`。
- `battle.py`: `snapshot()` 返回 `enemy_intent`。
- `Battle.vue`: 敌人头像旁显示意图徽记。

#### 成熟策划评估

| 维度 | 评估 |
|---|---|
| 用户爽感 | 高。玩家会觉得自己“读懂并反制” |
| 策略深度 | 高 |
| 开发成本 | 中 |
| 风险 | 意图必须可靠,不能显示 A 实际做 B |
| 优先级 | P0 |

### 3.5 连招与斩杀

#### 功能目标

把卡牌从“单张按钮”升级成“组合爽点”。

#### 连招设计

以现有技能为基础,给技能加 `tags`:

| 标签 | 示例技能 | 作用 |
|---|---|---|
| focus | 凝神咒、剑意凝聚 | 强化下一击 |
| speed | 御风诀、夜行如风 | 提高命中/先手 |
| guard | 护体罡气、根基诀 | 降低下回合风险 |
| strike | 剑诀·初、齿轮诀 | 主要伤害 |
| ult | 沧澜九式、天机轮 | 终结爆发 |

示例连招:

| 连招 | 条件 | 效果 |
|---|---|---|
| 凝神破绽 | focus -> strike | 下一击额外 +20% 暴击伤害 |
| 御风追击 | speed -> 低命中攻击 | 命中补正 +15% |
| 剑意斩杀 | focus -> ult 且敌 HP < 30% | 触发斩杀演出 |
| 稳守反击 | guard 后受到攻击 | 下回合攻击 +25% |

#### 斩杀设计

触发条件:

```text
敌方 HP <= 15%
或 卡牌预计伤害 >= 敌方当前 HP
或 连招进入终结态
```

UI 表现:

- 卡牌出现“可斩杀”标记。
- 敌人血条出现斩杀线。
- 点击后显示短终结演出。
- 战后高光记录“斩杀回合”。

#### 当前框架落点

- `skill_templates.py`: 增加 `tags` / `combo_role`。
- `battle.py`: 在 `state.buffs` 或新增 `combo_state` 里记录上一招。
- `Battle.vue`: 卡牌预览已经有预计伤害,可加斩杀判定。

#### 成熟策划评估

| 维度 | 评估 |
|---|---|
| 用户爽感 | 极高。连招与斩杀是点击爽感核心 |
| 策略深度 | 中-高 |
| 开发成本 | 中 |
| 风险 | 数值容易爆炸,需要保守加成 |
| 优先级 | P0 |

### 3.6 战后高光总结

#### 功能目标

每场战斗结束后,玩家必须知道自己得到了什么、推进了什么、下一步该做什么。

#### 结算页结构

```text
胜负结果
  -> 本战高光:最高伤害 / 暴击 / 连招 / 斩杀
  -> 奖励:修为、灵气、材料、心得
  -> 推进:日课、图鉴、Boss 线索、配方进度
  -> 下一步:继续刷同族 / 去炼制 / 精进技能 / 看本命书
```

#### 当前框架落点

- `Battle.vue` 已有结算页,扩展 `rewards` 和 `commit` 展示。
- `store.py` 已记录 kill/drop/daily,可返回 progress delta。
- `recipes.py` 有材料反向索引 `MATERIAL_USAGE`。

#### 成熟策划评估

| 维度 | 评估 |
|---|---|
| 用户爽感 | 高。让掉落和成长更有存在感 |
| 留存价值 | 极高。下一步按钮直接决定继续率 |
| 开发成本 | 中 |
| 风险 | 信息太多会乱,只展示本战相关 |
| 优先级 | P0 |

## 4. P1 功能:成长与构筑

### 4.1 法宝装备系统

当前 `items.py` 已定义 `equipment` 和 `equip_stats`,但背包主要展示和使用消耗品,法宝还没有形成“穿戴后改变战斗”的强闭环。

#### 推荐装备槽

| 槽位 | 示例 | 定位 |
|---|---|---|
| 本命法宝 | 风暴之刃、百川剑 | 主属性和打法核心 |
| 护身法器 | 蛇鳞甲、狐尾护符 | 生存/闪避 |
| 灵宝 | 万象齿轮、开源秘卷 | 稀有规则改变 |
| 饰物 | 鹰眼护目镜、共情玉 | 暴击/反制 |

#### 成熟策划判断

装备系统是“掉落有意义”的关键。如果玩家打完怪只看到材料数字,爽感会很快疲劳;如果材料能变成装备并改变打法,玩家会自然继续刷。

#### 当前框架落点

- `Inventory.vue`: 增加装备按钮和已装备状态。
- `store.py`: character 增加 `equipped_items`。
- `attributes.py`: `refresh_derived` 叠加装备属性。
- `battle.py`: 读取装备特殊词条。

#### 风险

- 装备属性不要直接线性堆太大,否则技能树和境界被稀释。
- 稀有装备要给规则型效果,不要只给大数字。

优先级: P1。

---

### 4.2 技能构筑与卡组压缩

当前已有 56 招、装备槽 6 个,这是非常好的卡组基础。下一步不是继续加招式,而是让玩家形成流派。

#### 门派流派建议

| 门派 | 核心爽感 | 主要机制 |
|---|---|---|
| 沧澜剑派 | 蓄力暴击、一剑斩杀 | focus、crit、execute |
| 天机阁 | 稳定连发、命中控制 | multi-hit、accuracy、draw |
| 玄机宗 | 高暴击和破局 | weakpoint、crit chain |
| 青冥派 | 护体反击、越打越稳 | guard、counter、regen |
| 月隐宫 | 闪避追击、低血翻盘 | evade、speed、revenge |

#### 卡组规则

- 默认 6 张上阵。
- 每个门派至少 2 套推荐卡组。
- 自动提示“当前卡组缺少终结技/恢复/防御”。
- 新手只看推荐,进阶玩家再自由调整。

#### 当前框架落点

- `Skills.vue`: 已有装备槽 UI。
- `skill_templates.py`: 增加 tag,用于推荐与连招。
- `battleApi.cardPreview`: 增加 combo 后预计收益。

优先级: P1。

### 4.3 境界突破仪式

当前升级和境界变化已有数据基础,但仪式感不足。由于本游戏把模型档位和境界绑定,突破是最该做成高光的点。

#### 触发时机

- 达到新境界等级阈值。
- Key 已验证该境界所需模型。
- 若模型不可用,显示“灵脉不足,需更换 Key 或暂缓突破”。

#### 仪式内容

```text
突破预告 -> 消耗破境丹/材料可选 -> 天劫三选一 -> 成功 -> 模型升档展示 -> 本命书破境章
```

#### 关键爽点

- 显示旧模型 -> 新模型。
- 解锁新技能。
- 属性大幅刷新。
- 生成“破境章”。

#### 当前框架落点

- `sects.py`: 已有境界模型梯度。
- `attributes.py`: 已有 realm/level 属性派生。
- `cultivation.py`: 可生成破境章。
- `Initiation.vue` 的仪式动画可复用一部分视觉语言。

优先级: P1。

### 4.4 推荐目标系统

#### 功能目标

解决“我现在该做什么”的问题。

#### 推荐目标类型

| 类型 | 示例 |
|---|---|
| 材料目标 | 再击败 2 只狐妖可炼制狐尾护符 |
| 技能目标 | 还差 1 个招式心得·初可精进剑诀·初 |
| 图鉴目标 | 山林狐妖族研究 4/5,再首杀三尾灵狐 |
| Boss 目标 | 收集 3 条线索后解锁墨魔·林泽 |
| 生存目标 | 当前 HP 偏低,建议打坐或使用丹药 |

#### 展示位置

- 主城顶部: 1 个主推荐。
- 探索地图: 每只怪 hover 展示“击败后推进什么”。
- 战后结算: 只展示本战相关下一步。

优先级: P1。

## 5. P2 功能:秘境 Roguelite

### 5.1 为什么适合《灵枢笔录》

当前主循环是:

```text
地图点怪 -> 战斗 -> 掉落 -> 回地图
```

这适合作为日常刷怪,但不足以制造“再来一局”。Roguelite 秘境能把 5-10 分钟体验包装成完整一局:

```text
进入秘境 -> 节点选择 -> 战斗/奇遇/商店/祝福 -> 小 Boss -> 总结章
```

它能复用现有战斗、物品、奇遇、墨炉,开发性价比高。

### 5.2 秘境结构

| 节点 | 占比 | 作用 |
|---|---:|---|
| 战斗 | 50% | 高频爽感和掉落 |
| 奇遇 | 20% | 随机变化 |
| 商店/炼制 | 10% | 消耗材料和选择 |
| 祝福 | 15% | 构筑变化 |
| Boss | 5% | 局末高光 |

### 5.3 祝福示例

| 祝福 | 效果 |
|---|---|
| 剑心通明 | focus 后下一击暴击率 +20% |
| 灵气回涌 | 击杀后回复 15% 灵气 |
| 破甲真言 | 每回合首次攻击降低敌人防御 |
| 山海眷顾 | 本局掉落数量 +1 |
| 墨炉余温 | 战后成章 token 修为 +10% |

### 5.4 Token 策略

因为本项目坚持玩家自带 Key,秘境必须有清晰成本选项:

| 选项 | 说明 |
|---|---|
| 极速秘境 | 节点战斗极速结算,局末可选择总结章 |
| 正传秘境 | 精英和 Boss 生成叙事,普通节点极速 |
| 全章秘境 | 每个节点都成章,明确高消耗 |

默认推荐“正传秘境”,兼顾特色与成本。

### 5.5 当前框架落点

- 新后端模块: `backend/app/realms.py` 或 `dungeons.py`
- 新前端页面: `frontend/src/views/RealmRun.vue`
- 复用: `battle.py`, `events.py`, `recipes.py`, `cultivation.py`
- 存档字段: `active_realm_run`, `realm_history`

优先级: P2。

## 9. 推荐开发路线

### Sprint 1: 接入可信 + 第一局爽感

目标: 让新用户带 Key 完成第一场战斗并知道下一步。

范围:

- 修正 Key 存储文案与实际实现。
- Onboarding 加成本契约和预算提示。
- 新手三步入道。
- 探索地图增加推荐第一只怪。
- 战后结算增加“下一步”按钮。

验收:

- 新用户验证 Key 后 3 分钟内能完成第一战。
- 第一战后至少有一个明确强化入口。

### Sprint 2: 高频战斗爽感

目标: 小怪战变快,出牌变有判断。

范围:

- 普通小怪默认极速模式。
- 首遇/精英/Boss 默认正传模式。
- 敌人意图。
- 连招状态。
- 斩杀线和终结按钮。
- 战后高光统计。

验收:

- 普通小怪单局平均 10-25 秒。
- 玩家能在出牌前看见敌方意图。
- 结算页能展示至少 3 类高光或进度。

### Sprint 3: 掉落到成长闭环

目标: 让材料、技能、法宝有明确用途。

范围:

- 装备槽和法宝穿戴。
- 背包材料“可用于”跳转。
- 技能流派标签和推荐卡组。
- 当前目标推荐。
- 破境仪式第一版。

验收:

- 玩家拿到材料后能看到它推进哪个配方。
- 玩家能装备法宝并在战斗属性里看到变化。
- 境界提升有全屏仪式和本命书记录。

### Sprint 4: 中期玩法扩展

目标: 做出“再来一局”的 5-10 分钟玩法。

范围:

- 秘境 Roguelite。
- 祝福系统。
- 秘境局末总结章。
- 山海经研究等级。
- Boss 线索解锁。

验收:

- 秘境一局 5-8 个节点。
- 局内至少 3 次有意义选择。
- 局末总结能接回装备/技能/图鉴目标。

## 11. 成熟策划风险评估

### 11.1 BYOK 信任风险

这是最高风险。用户必须交 Key,所以任何模糊文案都会被放大。

应对:

- 文案与技术实现一致。
- 给删除/更换能力。
- 给精确 usage 或估算标识。
- 预算提醒默认打开。

### 11.2 LLM 等待风险

如果每回合都等模型,爽感会被消耗。

应对:

- 数值即时。
- 小怪极速。
- 叙事后台。
- 首遇/Boss/突破保留仪式感。

### 11.3 数值膨胀风险

装备、技能、连招、境界都加伤害,很容易爆表。

应对:

- 连招优先给命中、暴击、斩杀演出,少给直接乘区。
- 装备多做规则型效果。
- Boss 用机制和意图增加难度,不要只堆血。

### 11.4 内容疲劳风险

113 怪如果只是不同名字,玩家会很快疲劳。

应对:

- 每族绑定不同意图和掉落用途。
- 图鉴研究给弱点和故事。
- Boss 线索把小怪变成主线入口。

### 11.5 UI 信息过载风险

当前主城信息已经偏密,再加系统可能更乱。

应对:

- 主城只显示 1 个主推荐和 3 个次入口。
- 战后只显示本战相关进度。
- 高级系统折叠到“更多”或专门页面。

## 12. 最终判断

从成熟游戏策划角度看,《灵枢笔录》不缺概念,也不缺内容量。它下一阶段最需要的是把“概念爽”落到“操作爽”:

```text
出招要爽
结算要爽
变强要爽
掉落要有去处
Key 消耗要有价值感
```

最值得优先做的不是大而全的新系统,而是这条闭环:

```text
Key 接入可信
  -> 第一场战斗快
  -> 出牌有敌人意图
  -> 连招/斩杀有高光
  -> 掉落立刻指向装备/技能/炼制
  -> 战斗沉淀进本命书/图鉴/Boss 线索
```

这条线打通后,游戏会从“AI 修仙概念 demo”变成“用户愿意继续刷、继续写、继续突破”的产品。

# 《灵枢笔录》全局数值分析与优化方案

> 版本: v0.1  
> 目标: 在“不设置氪金项目、不设置修为上限、成长围绕 token 消耗”的前提下,重建一套可持续扩展、战斗不拖沓、怪物/Boss/NPC 难度可控的数值体系。

## 1. 核心结论

当前项目的数值问题不是单个怪物血量填高了,而是几套成长曲线叠加在一起后发生了系统性膨胀:

1. 玩家属性使用 `derive_stats(attrs, level)` 计算,生命、攻击、防御、灵气都会乘 `realm_multiplier(level)`。
2. 普通怪物表里的 `hp/atk/def` 已经按怪物阶位逐步变大,进入战斗时又在 `BattleEngine` 中再次乘 `realm_multiplier(enemy.level)`。
3. Boss 表面上在接口里直接展示配置值,但进入战斗时也会被 `BattleEngine` 乘等级倍率,导致 Boss 真实战斗血量远高于展示血量。
4. NPC 弟子在生成时已经按 `realm_multiplier(npc_level)` 算过最终属性,随后被包装成临时 `Enemy`,进战斗后再次乘等级倍率,属于最严重的“双重膨胀”。
5. 当前伤害公式 `atk * power * 100 / (100 + def)` 在防御随等级一起膨胀后,会让高等级双方的实际伤害趋近于一个很小的常数,但怪物血量仍然随等级大幅上涨,所以战斗回合数会爆炸。
6. 灵气既是战斗资源又被当作交易货币,而灵气上限随等级巨大膨胀,导致中后期道具价格、丹药恢复、技能消耗都失去意义。

数值优化的主线应该是:

- 修为/等级继续由 token 真实消耗驱动,不设置上限。
- 战斗难度改为“目标回合数驱动”,普通怪 3-6 回合,精英怪 5-9 回合,神话怪 8-14 回合,Boss 10-20 回合。
- 怪物/Boss/NPC 的最终战斗属性不再直接用静态表值乘等级倍率,而是以“同级玩家基准属性 × 怪物阶位系数 × 族群修正”生成。
- 伤害公式保持不变,怪物 `hp/atk/def_` 改为围绕目标血量比、目标伤害比反推。
- 技能消耗、丹药恢复、装备属性、商店价格全部改为百分比或随等级缩放,避免中后期失效。

## 2. 当前数值结构

### 2.1 角色成长

当前角色主要由 8 属性派生:

| 属性 | 当前作用 |
| --- | --- |
| 力 str | 攻击 |
| 元 qi | 灵气上限 |
| 躯 vit | 生命上限 |
| 敏 agi | 速度、闪避 |
| 意 wis | 暴击率、暴击伤害 |
| 韧 end | 防御、疲劳上限 |
| 缘 fate | 运气、掉落倾向 |
| 悟 ins | 章节命题、历史兼容修为倍率 |

核心公式在 `backend/app/attributes.py`:

```python
realm_multiplier(level) = 1.0 + level * 0.2 + (level / 10) ** 2.8

atk    = (20 + str * 2.5) * realm_multiplier(level)
max_qi = (600 + qi * 30) * realm_multiplier(level)
max_hp = (80 + vit * 8) * realm_multiplier(level)
spd    = (75 + agi * 1.5) * realm_multiplier(level)
def    = (8 + end * 1.2) * realm_multiplier(level)
```

升级规则:

```python
next_level_need = (current_level + 1) * 100
每升 1 级: 随机属性 +2
升级时: HP/QI 回满
```

这个等级曲线在“每次战斗 token 消耗较低”的假设下看似可用,但实际 LLM 战斗经常单场消耗数千 token,因此会出现“一场怪物直接升级甚至连升”的问题。修为升级曲线应以后续方案为准: [《灵枢笔录》修为、等级卡点与三条资源重规划](./CULTIVATION_LEVEL_PROGRESSION_REDESIGN.md)。

旧曲线从 Lv.1 到关键等级的大致累计消耗为:

| 等级 | 累计 token 约值 |
| --- | ---: |
| Lv.10 | 5,400 |
| Lv.30 | 46,400 |
| Lv.60 | 182,900 |
| Lv.100 | 504,900 |
| Lv.160 | 1,287,900 |
| Lv.200 | 2,009,900 |
| Lv.300 | 4,514,900 |

这个旧曲线不再建议直接使用。保留此处仅用于说明当前实现为什么会导致升级过快。需要注意两点:

- 每级只随机加 2 点属性,玩家可能长期随机不到 `vit/end`,出现“同级但血防偏低”的挫败。
- 200 级以后虽然没有硬上限,但门派模型阶段、怪物表、Boss 表都没有无上限对应策略。

### 2.2 普通怪物

普通怪物配置在 `backend/app/enemies.py`,每个怪物静态配置 `level/hp/atk/def/spd/evasion/rewards/drops`。接口与战斗中都会计算:

```python
enemy_final_hp  = enemy.hp  * realm_multiplier(enemy.level)
enemy_final_atk = enemy.atk * realm_multiplier(enemy.level)
enemy_final_def = enemy.def * realm_multiplier(enemy.level)
```

这导致怪物静态表值越高,越会被等级倍率放大。

当前样本:

| 等级 | 玩家生命 | 怪物 | 怪物生命 | 怪物/玩家生命 |
| ---: | ---: | --- | ---: | ---: |
| Lv.1 | 135 | 山林小狐 Lv.3 | 130 | 1.0x |
| Lv.10 | 614 | 双尾赤狐 Lv.10 | 800 | 1.3x |
| Lv.30 | 5,548 | 五尾媚妖 Lv.30 | 37,276 | 6.7x |
| Lv.60 | 41,560 | 七尾狐圣 Lv.60 | 1,229,599 | 29.6x |
| Lv.100 | 217,427 | 青龙 Lv.100 | 22,818,507 | 104.9x |
| Lv.160 | 1,081,839 | 盘古残魂 Lv.160 | 310,119,450 | 286.7x |

按等级段看,普通怪物最终生命与同级玩家生命的中位比值:

| 等级段 | 当前中位生命比 |
| --- | ---: |
| Lv.1-15 | 0.9x |
| Lv.16-30 | 3.8x |
| Lv.31-45 | 10.5x |
| Lv.46-65 | 24.9x |
| Lv.66-85 | 47.2x |
| Lv.86-105 | 87.4x |
| Lv.106-125 | 143.1x |
| Lv.126-150 | 198.3x |
| Lv.151+ | 275.6x |

结论: 15 级前体验基本合理,30 级后开始明显拖沓,60 级后战斗已经不符合可玩标准。

### 2.3 Boss

Boss 配置在 `backend/app/bosses.py`。`boss_to_dict()` 展示的是静态值,但战斗使用 `get_enemy()` 返回 Boss 后,仍在 `BattleEngine` 里乘等级倍率。

样本:

| Boss | 等级 | 配置血量 | 战斗实际血量 | 同级玩家生命比 |
| --- | ---: | ---: | ---: | ---: |
| 墨魔·林泽 | 40 | 8,000 | 460,023 | 37.5x |
| 拥抱真人 | 78 | 25,000 | 8,281,922 | 86.4x |
| 阿莫迪克斯 | 145 | 180,000 | 326,842,214 | 425.0x |
| 界外神祇 | 180 | 300,000 | 992,586,361 | 607.9x |

结论: Boss 不应该依靠 400-600 倍血量制造史诗感,而应该依靠阶段机制、招式变化、叙事演出、奖励和章节重要性制造压迫感。

### 2.4 NPC 弟子

NPC 在 `backend/app/npc.py` 生成时已经按等级倍率计算:

```python
final_mult = realm_multiplier(npc_level) * rank_stat_mult * 0.8
hp  = base_hp  * final_mult * 4
atk = base_atk * final_mult
def = base_def * final_mult
```

随后 `_start_npc_battle()` 又把这个最终值塞进临时 `Enemy`,而 `BattleEngine` 再次执行:

```python
enemy_hp = enemy.hp * realm_multiplier(enemy.level)
```

结论: NPC 切磋/真斗的实际战斗属性会被等级倍率计算两次。这个必须作为 P0 修复。

### 2.5 战斗伤害

当前玩家伤害:

```python
damage = player_atk * skill_power * 100 / (100 + enemy_def)
crit: damage *= player_crit_dmg
```

当前敌人伤害:

```python
damage = enemy_atk * enemy_skill_power * 100 / (100 + player_def)
crit: damage *= 1.6
```

问题在于 `atk` 和 `def` 都乘同一个等级倍率。等级越高,`100 + def` 中的 `100` 越不重要,伤害大约会退化为:

```text
damage ≈ atk_base / def_base * skill_power * 100
```

也就是说,等级越高,攻击增长越难反映到真实伤害上;但怪物生命仍然指数式膨胀,于是玩家每回合伤害几乎不变,怪物血量却越来越离谱。

当前估算回合数:

| 等级 | 怪物 | 玩家预估每击伤害 | 玩家击杀回合 | 敌人击败玩家回合 |
| ---: | --- | ---: | ---: | ---: |
| Lv.10 | 双尾赤狐 | 182 | 4.4 | 10.9 |
| Lv.30 | 五尾媚妖 | 583 | 63.8 | 18.5 |
| Lv.60 | 七尾狐圣 | 563 | 2,180.6 | 55.1 |
| Lv.100 | 青龙 | 381 | 59,792.8 | 75.0 |
| Lv.160 | 盘古残魂 | 660 | 469,777.0 | 149.1 |

结论: 当前战斗不是“难”,而是“玩家几乎打不动”。这会直接造成用户认为页面卡住、战斗没有反馈。

### 2.6 技能与心法

当前招式系统在 `backend/app/skill_templates.py`:

- power: 1.0 为基础攻击。
- max_level = 5。
- 每级 power +8%。
- 每级 qi_cost -4%,最低 80%。
- 每级 crit_bonus +1%。
- 升级消耗: 初/中/高/绝心得分别用于 Lv1->2、Lv2->3、Lv3->4、Lv4->5。

主要问题:

1. 高等级灵气上限会达到几十万甚至上百万,但技能消耗仍然是 8、20、60、300 这种固定值,中后期没有资源压力。
2. 多数 buff 技能描述没有真实效果。当前 `_compute_outcome()` 里只要 `card.type == "buff"`,统一处理成“下一击 ATK +50%”。例如“护体罡气”“御风诀”“剑意凝聚”“月隐之姿”等都没有按描述实现。
3. 心法升级的 +8% power 是合理的,但在当前怪物血防曲线下完全救不回来。
4. 命中率从 `card.hit_rate - enemy_evasion` 得出。高闪避怪配低命中 ult 时,命中可低到 40%-50%,但战斗又极长,会放大挫败感。

### 2.7 怪物招式

怪物招式在 `backend/app/monster_skills.py`:

| 等级段 | 招式池权重 |
| --- | --- |
| Lv.1-30 | basic 70%, mid 30% |
| Lv.31-70 | basic 40%, mid 40%, high 20% |
| Lv.71-100 | basic 20%, mid 30%, high 35%, ult 15% |
| Lv.100+ | mid 20%, high 40%, ult 40% |

怪物技能倍率从 0.9 到 3.1。这个设计本身可用,但必须建立在怪物 `atk` 合理的基础上。当前高阶怪物 `atk` 被倍率放大后,再叠 high/ult 技能,会出现单次伤害跳变过大。

### 2.8 掉落

当前普通掉落概率:

```python
rarity_probs = {
  1: 80%,
  2: 50%,
  3: 25%,
  4: 10%,
  5: 3%,
  6: 0.5%
}
```

技能心得掉落已经改为按怪物等级和 tier 增长:

| tier | 当前心得掉落 |
| --- | --- |
| low | 约 20%-36% |
| mid | 约 28%-72% |
| high | 约 78%-98% |
| myth | 必掉 |

问题:

1. 普通材料掉落概率只看物品稀有度,不看怪物 tier,低阶怪和高阶怪对同稀有度材料的掉率逻辑差异不够。
2. `drop_pity` 是全局稀有保底,没有按物品池、怪物阶位、稀有度分层。玩家可能用低价值怪物堆保底,再转打高价值怪。
3. 多个 drops 独立判定,高阶怪可能一次掉很多,但战斗又极长,整体奖励节奏不稳定。

### 2.9 道具、装备与经济

当前道具数值在 `backend/app/items.py`:

- 凝神丹: 灵气 +100。
- 灵元丹: 灵气 +200。
- 回元丹: HP +30%,灵气 +300。
- 上品回春丹: HP +100%。
- 装备多为固定值,如攻击 +120、攻击 +250、灵气 +200、防御 +12%。
- NPC 商店用角色当前 `qi` 作为支付货币。

问题:

1. 固定灵气恢复在中后期失效。Lv.60 玩家灵气上限可超过 20 万,+300 等于几乎没有。
2. 固定攻击装备在中后期失效。Lv.60 玩家攻击已超过 1 万,+250 的边际收益极低。
3. 用战斗灵气当货币会让“战斗资源”和“经济资源”互相污染。高等级灵气上限巨大时,商店价格会变得没有意义;低血低蓝时买东西又会影响战斗手感。

### 2.10 打坐、疲劳与墨炉

打坐当前设计:

```python
3 秒内连续点击形成 streak
基础恢复: 1% max_hp + 1% max_qi -1 疲劳
连击倍率最高约 5x
打坐本身不直接加修为,入定/闭关章节进入墨炉后通过 token 增长修为
```

这个方向符合产品目标: 打坐用于恢复状态,入定/闭关用于消耗 token 成章并转修为。

但当前疲劳系统几乎只被奇遇影响,战斗、长时间闭关、连续入定没有形成完整闭环。建议疲劳作为“限制连续战斗效率,推动打坐恢复”的软约束,而不是硬性阻止玩家。

## 3. 优化目标

### 3.1 战斗节奏目标

| 内容 | 目标回合数 | 设计感受 |
| --- | ---: | --- |
| 教学怪 | 2-4 回合 | 让用户立刻理解出招和胜利 |
| 普通同级怪 | 3-6 回合 | 快速、有掉落反馈 |
| 高血防怪 | 5-8 回合 | 明显更硬,但不拖 |
| 高攻低血怪 | 3-5 回合 | 有压迫感,容错低 |
| 高阶精英 | 6-10 回合 | 需要选技能和补给 |
| 神话怪 | 8-14 回合 | 小 Boss 体验 |
| 主线 Boss | 10-18 回合 | 有阶段和叙事演出 |
| 终局 Boss | 15-25 回合 | 允许长,但必须有阶段变化 |

任何单场战斗预估超过 25 回合,都应该触发系统性压缩:

- 怪物进入“破绽”状态,每回合受到伤害 +8%,最多 +80%。
- 或玩家本命书共鸣,每 5 回合获得一次免费天命能量。
- 或允许“自动推演”快速结算低风险回合。

### 3.2 难度目标

| 难度 | 怪物等级 | 血量系数 | 攻击系数 | 奖励 |
| --- | --- | ---: | ---: | --- |
| 轻松 | 玩家等级 -8 到 -3 | 0.6-0.9x 玩家生命 | 0.45-0.6x 玩家攻击 | 低 |
| 同级 | 玩家等级 -2 到 +2 | 0.9-1.4x 玩家生命 | 0.6-0.8x 玩家攻击 | 标准 |
| 精英 | 玩家等级 +3 到 +6 | 1.8-3.0x 玩家生命 | 0.75-1.0x 玩家攻击 | 高 |
| 危险 | 玩家等级 +7 到 +10 | 3.0-5.0x 玩家生命 | 0.95-1.2x 玩家攻击 | 很高 |
| 神话 | 剧情/高阶地图 | 5.0-8.0x 玩家生命 | 1.1-1.35x 玩家攻击 | 稀有 |
| Boss | 明确入口 | 8.0-20.0x 玩家生命 | 1.2-1.5x 玩家攻击 | 章节/稀有掉落 |

注意: “攻击系数”不是最终每回合扣血比例,还要经过防御、命中、技能倍率和暴击。

### 3.3 经济目标

1. 战斗消耗有意义: 高阶招式不应无限释放。
2. 药品始终有价值: 低级药前期好用,高级药中后期仍有效。
3. 装备始终有价值: 固定值装备适合前期,高阶装备必须转百分比或随等级成长。
4. token 修为不可被掉落、战斗直接奖励绕开: 战斗只创建章节任务,修为由墨炉 token 消耗结算。
5. 不设置修为上限: 通过成长曲线和内容层级扩展,不通过硬上限截断。

## 4. 推荐新数值框架

### 4.1 拆分三层数值

建议新增 `backend/app/combat_balance.py`,把数值分成三层:

1. 角色成长层: 继续由 `attributes.py` 负责,用于展示玩家成长、等级、境界和基础战斗属性。
2. 战斗平衡层: 由 `combat_balance.py` 根据玩家等级生成怪物/Boss/NPC 的最终战斗属性。
3. 经济缩放层: 由 `economy_balance.py` 或同文件函数处理道具价格、恢复量、装备缩放。

不要再让普通怪、Boss、NPC 各自写一套“看起来像最终值”的属性,再进入战斗统一乘倍率。

### 4.2 玩家基准属性

为了生成同级怪物,需要一个“同级玩家基准”。推荐:

```python
def expected_attr(sect_base: int, level: int) -> float:
    bless_avg = 7.5 / 8
    level_avg = max(0, level - 1) * 2 / 8
    return sect_base + bless_avg + level_avg

def expected_player_stats(sect_id: str, level: int) -> dict:
    attrs = {
        key: expected_attr(base_value, level)
        for key, base_value in SECT_INITIAL_ATTRS[sect_id].items()
    }
    stats = derive_stats(attrs, level)
    tier = get_tier_for_level(sect_id, level)
    stats["battle_atk"] = int(stats["atk"] * (tier.atk_multiplier if tier else 1.0))
    return stats
```

这个基准不是玩家真实属性,而是用于生成怪物难度的“同级平均值”。真实玩家因为资质、心法、装备、技能升级而上下浮动。

### 4.3 怪物最终属性生成

推荐不要直接使用:

```python
enemy.hp * realm_multiplier(enemy.level)
```

改为:

```python
base = expected_player_stats(player_sect, enemy.level)
profile = ENEMY_TIER_PROFILE[enemy.tier]
clan = CLAN_PROFILE[enemy.clan]

enemy_max_hp = base["max_hp"] * profile.hp * clan.hp
enemy_atk    = base["battle_atk"] * profile.atk * clan.atk
enemy_def    = base["def_"] * profile.def_ * clan.def_
enemy_spd    = base["spd"] * profile.spd * clan.spd
```

推荐 tier 系数:

| tier | HP | ATK | DEF | SPD | 定位 |
| --- | ---: | ---: | ---: | ---: | --- |
| low | 0.8 | 0.55 | 0.55 | 0.95 | 普通低威胁 |
| mid | 1.4 | 0.70 | 0.75 | 1.00 | 标准怪 |
| high | 2.6 | 0.90 | 0.95 | 1.05 | 精英怪 |
| myth | 5.5 | 1.10 | 1.10 | 1.08 | 神话怪 |
| boss_story | 8.0 | 1.15 | 1.10 | 1.00 | 普通 Boss |
| boss_arc | 14.0 | 1.25 | 1.20 | 1.05 | 章节 Boss |

1. 作为历史兼容字段,暂时不删除。
2. 作为“个体偏移权重”,例如同 tier 同族中,血量高的怪获得 `+0%-20%` 的 HP 个体修正。

```python
rank_index = clan_member_index / max(1, clan_size - 1)
```

### 4.6 NPC 修正

P0 必须修复 NPC 双重倍率。两种方案:

## 5. 伤害比例优化

### 5.1 公式原则

不再替换伤害公式。为了让玩家建立稳定直觉,战斗继续使用既有公式:

```python
damage = attacker_atk * skill_power * 100 / (100 + defender_def)
```

暴击:

```python
if crit:
    damage *= crit_dmg
```

新的平衡方式不是改公式,而是反推怪物属性:

1. 先确定敌人血量占玩家血量的比例,例如同级普通怪 0.8x-1.2x,精英怪 1.3x-2.0x,神话怪 2.0x-3.2x。
2. 再确定玩家每次命中应打掉敌人多少比例,例如普通怪 35%-55%,精英怪 20%-30%,神话怪 14%-18%。
3. 根据固定伤害公式反推出敌人的 `def_`。
4. 再确定敌人每次命中应打掉玩家多少比例,根据固定伤害公式反推出敌人的 `atk`。

这样玩家理解的公式不变,但血量和伤害都保持比例平衡。

### 5.2 命中与闪避

推荐:

```python
hit_rate = clamp(card.hit_rate - defender_evasion + attacker_accuracy, 0.65, 0.98)
```

怪物命中:

```python
enemy_hit = clamp(skill.hit_rate - player_evasion, 0.60, 0.96)
```

连续 miss 保护:

```python
if player_miss_streak >= 2:
    next_hit_rate += 0.15
```

目的不是取消闪避,而是避免玩家连续 3-4 次失手后觉得战斗卡住。

### 5.3 回合压缩机制

如果进入战斗时预估 TTK 超过目标:

```python
estimated_turns = enemy_hp / expected_player_damage
if estimated_turns > target_turns * 1.8:
    enemy_hp = expected_player_damage * target_turns * random(0.9, 1.1)
```

或者更有仙侠味:

```python
round >= 8: 破绽 +8% 受伤
round >= 12: 破绽 +24% 受伤
round >= 16: 破绽 +48% 受伤
```

这样即使数值表有误,战斗也不会无限拖。

## 6. 技能与心法优化

### 6.1 技能消耗改为百分比

当前固定 qi_cost 中后期失效。推荐:

| 技能 tier | 当前典型消耗 | 新消耗 |
| --- | ---: | --- |
| basic | 8-20 | 1% max_qi,最低 8 |
| normal | 10-25 | 2% max_qi,最低 15 |
| special | 20-40 | 4% max_qi,最低 25 |
| ult | 55-300 | 7% max_qi,最低 60 |
| destiny | 无固定 | 15% max_qi 或天命槽 |

技能升级可以降低百分比:

```text
Lv.1: 100% 消耗
Lv.2: 94%
Lv.3: 88%
Lv.4: 82%
Lv.5: 76%
```

这样高等级玩家仍然需要管理灵气,但升级技能会明显变舒服。

### 6.2 Buff 技能必须按描述实现

当前所有 buff 都被统一成下一击 ATK +50%。建议拆成:

| buff 类型 | 实际效果 |
| --- | --- |
| 凝神咒 | 下一击伤害 +50% |
| 剑意凝聚 | 下一击暴击率 +30%,暴击伤害 +20% |

| 千古经文 | 3 回合防御倍率 +25% |
| 夜行如风 | 3 回合先手概率提升,闪避 +10% |
| 月隐之姿 | 2 回合无法被暴击 |

这会让卡牌选择真正有策略,也能配合你之前要求的技能卡悬浮提示。

### 6.3 心得消耗保留,但节奏分层

当前升级消耗:

| 升级 | 材料 |
| --- | --- |
| Lv1 -> Lv2 | 1 个初阶心得 |
| Lv2 -> Lv3 | 2 个中阶心得 |
| Lv3 -> Lv4 | 3 个高阶心得 |
| Lv4 -> Lv5 | 5 个绝阶心得 |

这个结构可以保留。建议补两条:

1. 高阶怪必掉对应心得的方向是合理的,但要防止玩家无法打动高阶怪。先修战斗数值,再评估心得产出。
2. 增加“心得拆解/合成”:

```text
3 初 -> 1 中
3 中 -> 1 高
3 高 -> 1 绝
1 绝 -> 2 高
```

这样低等级玩家也有长期目标,不会因为卡某个心得完全停住。

## 7. 怪物与 Boss 优化

### 7.1 普通怪定位

每个族群 9 个怪可以保留,但不再通过超大血量区分强弱。建议每族内部形成“机制递进”:

| 序号 | 定位 |
| --- | --- |
| 01-03 | 入门/普通怪,教学族群特色 |
| 04-06 | 中阶怪,开始出现机制 |
| 07-08 | 高阶精英,机制完整 |

- 七尾狐圣: 高闪避,但血量不能超过同级玩家 4 倍。
- 八尾妖王: 神话怪,可有一次“魅影替身”免伤,但血量控制在 6 倍左右。

### 7.2 Boss 不靠血厚,靠阶段

Boss 战建议:

1. 每个 Boss 绑定一个独特机制。
2. 每个阶段触发短叙事和技能池切换。
3. 奖励与本命书章节强绑定。

示例:

| Boss | 机制建议 |

| 界外神祇 | 四阶段: 凝视、降维、抹除、反证;每阶段改变命中/伤害规则 |

### 7.3 地图刷怪

当前地图按玩家等级 `span=20` 抽怪,对 60 级玩家可能刷出 80 级神话怪。建议改为分层池:

| 池 | 权重 | 等级范围 | 是否主动攻击 |
| --- | ---: | --- | --- |

| 危险 | 5% | player +7 到 player +10 | 明显红色警示 |

危险怪点击前必须显示“危险”状态,避免玩家误触进入不可打的战斗。

## 8. 掉落与奖励优化

### 8.1 心得掉落

保留当前方向:

```text
低阶怪: 初阶心得 20%-36%
中阶怪: 中阶心得 30%-72%
高阶怪: 高阶心得 78%-98%
```

不要在每个招式卡上重复写获取来源。

### 8.2 普通材料掉落

推荐按怪物 tier 分层:

| 怪物 tier | 白 | 绿 | 蓝 | 紫 | 橙 | 红 |
| --- | ---: | ---: | ---: | ---: | ---: | ---: |
| low | 70% | 25% | 6% | 0% | 0% | 0% |

| high | 40% | 70% | 35% | 12% | 3% | 0% |

- 精英怪最多 3 个。
- 神话怪保证 1 个核心材料 + 心得。
- Boss 保证 1 个专属掉落 + 章节任务。

### 8.3 保底机制

把当前全局 `drop_pity` 拆成:

```python
drop_pity = {
  "rare_material": 0,
  "epic_material": 0,
  "boss_unique": {},
}
```

规则:

- 蓝色材料保底: 6 次无蓝,下一次 +30%。
- 紫色材料保底: 12 次无紫,下一次 +25%。
- Boss 专属: 每个 Boss 独立保底,3 次未出则第 4 次必出。

这样不会出现“刷低级怪攒保底,再打高级怪吃保底”的问题。

### 8.4 战斗奖励

战斗不要直接给修为,这一点继续保持。推荐奖励分层:

| 战斗结果 | 立即奖励 | 延迟奖励 |
| --- | --- | --- |
| 胜利 | 少量灵气恢复、掉落、图鉴记录 | 正传章入墨炉,token 转修为 |
| 失败 | HP 降至 1,少量败战记录 | 败笔章入墨炉,仍可获得修为 |
| 撤退 | 保留剩余 HP/QI,无掉落 | 撤离章入墨炉 |
| 赠礼安抚 | 友好度、少量记录 | 人情章入墨炉 |
| NPC 切磋 | 友好度、教学/心得概率 | 切磋章入墨炉 |

不要让失败完全无收益,否则用户在高难战斗中会只感到浪费时间。失败章节照样消耗 token 并增长修为,这和产品方向一致。

## 10. 打坐、疲劳与 token 修为

### 10.1 打坐定位

打坐应该是“恢复状态 + 引导入墨炉”的入口,不是直接修为按钮。

当前“点击打坐恢复 HP/QI/疲劳”是正确方向。建议保留:

```text
每次点击: 恢复 1% HP/QI,降低疲劳
连击: 提升恢复倍率
打坐后提示: 可入定成章 / 闭关续写
```

### 10.2 疲劳闭环

推荐增加疲劳来源:

| 行为 | 疲劳变化 |
| --- | ---: |
| 普通战斗胜利 | +2 |
| 精英/神话战斗 | +4 |
| Boss 战 | +8 |
| 撤退 | +1 |
| 失败 | +5 |
| 入定成章 | +3 |
| 长闭关 | +10 |
| 打坐 | -1 到 -5 |

疲劳不要硬性阻止玩家,只做软影响:

| 疲劳比例 | 效果 |
| --- | --- |
| 0%-40% | 无影响 |
| 40%-70% | 战斗后 HP/QI 自然恢复降低 |
| 70%-90% | 怪物奇遇风险略增,打坐收益提升 |
| 90%+ | 出招消耗 +10%,但打坐连击更容易触发入定 |

这样疲劳会推动玩家打坐,打坐再推动入定消耗 token,形成产品闭环。

### 10.3 修为无上限

继续保持:

```python
char["exp"] += delta_tokens
check_level_up(char)
```

不设置等级上限。200 级以后建议:

1. 境界名继续扩展为“飞升一重、飞升二重、道祖、混元、无量”等。
2. 模型阶段没有更高档时继续复用最高 tier 模型。
3. 等级需求可以轻微提高,但不能硬卡:

```python
if level <= 200:
    need = (level + 1) * 100
else:
    need = int((level + 1) * 100 * (1 + (level - 200) * 0.004))
```

这不是上限,只是防止超高等级一天内冲穿所有数值内容。

5. 地图刷怪按安全/同级/精英/危险池生成。
6. 加入战斗回合压缩机制,防止异常数值导致超长战斗。

### P2: 经济与长期成长

1. 拆分灵气与灵石。
2. 重做商店价格。
3. 分层掉落保底。
4. 心得合成/拆解。
5. 200 级后境界命名和模型阶段复用策略。
6. 增加数值监控面板: 平均战斗回合、失败率、撤退率、token/level、心得产出。

## 12. 推荐验收标准

### 12.1 数值验收

每次改数值后输出一张 balance snapshot:

| 等级 | 玩家 HP | 玩家 ATK | 怪物 | 怪物 HP | 玩家击杀回合 | 怪物击杀回合 |
| ---: | ---: | ---: | --- | ---: | ---: | ---: |
| 10 | 自动生成 | 自动生成 | 同级 low | 自动生成 | 3-6 | 8-12 |
| 30 | 自动生成 | 自动生成 | 同级 mid | 自动生成 | 4-7 | 6-10 |
| 60 | 自动生成 | 自动生成 | 同级 high | 自动生成 | 6-10 | 5-9 |
| 100 | 自动生成 | 自动生成 | 同级 myth | 自动生成 | 8-14 | 4-8 |
| 160 | 自动生成 | 自动生成 | 高阶 myth | 自动生成 | 8-16 | 4-8 |

### 12.2 体验验收

- 普通战斗 10 秒内能看到明显进展。
- 任意战斗 3 回合内,敌我双方血条都必须有可见变化。
- 连续 miss 不超过 2 次。
- 玩家看到的技能预估伤害与真实伤害误差控制在 ±20%。
- Boss 每 30 秒至少出现一次阶段反馈或明显事件。
- 切磋不会出现“同级 NPC 血量远高于玩家数十倍”的情况。

### 12.3 经济验收

- Lv.1、Lv.30、Lv.60、Lv.100 的丹药恢复都有感知价值。
- 同品阶装备在解锁等级段内至少提供 5%-15% 的可感知提升。
- 高阶技能连续释放 5-8 次后会明显消耗灵气。
- 心得掉落能让活跃玩家稳定升级常用技能,但无法瞬间满所有技能。

## 14. 总结

这轮数值优化的关键不是单纯把怪物血量砍低,而是建立一套“可预测、可验收、可无限扩展”的数值生成体系。

当前最危险的问题是:

- 怪物、Boss、NPC 被等级倍率重复放大。
- 怪物属性没有围绕既有伤害公式做比例反推。
- 技能消耗和道具恢复没有随等级缩放。
- 灵气同时承担战斗资源和货币职责。

优先完成 P0 后,游戏会从“中后期完全打不动”恢复到“每个等级段都有合理战斗节奏”。随后再做 P1/P2,才能让 token 修为、本命书章节、打坐恢复、技能心得、掉落经济真正形成闭环。

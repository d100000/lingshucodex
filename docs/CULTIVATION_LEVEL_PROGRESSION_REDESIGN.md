# 《灵枢笔录》修为、等级卡点与三条资源重规划

> 日期: 2026-05-27  
> 目标: 解决“打一场怪物就升级、没有修为卡点、破境没有质变、血条/蓝条/疲劳成长失控或无感”的问题。  
> 适用范围: 当前 Web 版《灵枢笔录》的主角成长、战斗奖励、破境门槛、基础数值派生。  

> 主线副本、打怪动机、破境心魔与四宗归盟节奏的整合方案见: [《灵枢笔录》等级卡点 × 主线副本 × 小说化留存方案](./LEVEL_GATED_MAIN_STORY_DUNGEON_CAMPAIGN.md)。

---

## 0. 核心结论

当前最大的问题不是“给得太多”,而是升级系统把 `token` 直接等同于 `exp`:

```text
一场战斗消耗 token -> 直接进入 exp -> exp 达标 -> 连续升级
```

这会带来 4 个体验问题:

1. 玩家升级过快,没有修仙题材应有的“卡境界”感。
2. 战斗消耗的文本越长,玩家升级越快,数值会被 AI 输出长度影响。
3. 每级提升太平均,破境没有“从凡修到筑基”的质变。
4. 血条、蓝条、疲劳没有承担明确玩法职责,只是跟着等级变大。

新的成长设计应改为:

```text
原始 token 仍然记入总修为账本。
但升级只吃“有效修为”,有效修为需要经过战斗风险、怪物等级、首胜、任务奖励、破境材料共同结算。
```

也就是说:

```text
token 是修行痕迹;
有效修为才是能推动等级突破的力量。
```

如果短期不想改奖励结构,也至少要把等级需求曲线拉到“单场普通战斗不能升级”的量级。

## 1. 设计原则

### 1.1 升级节奏

推荐节奏:

| 阶段 | 玩家感受 | 单级目标时长 | 主要成长来源 |
|---|---|---:|---|
| Lv.1-10 炼气 | 快速熟悉,但不能一战一升 | 3-8 场有效战斗 | 普通怪、首胜、入门任务 |
| Lv.11-25 筑基 | 开始形成构筑和门派路线 | 6-15 场有效战斗 | 精英怪、悬赏、低级秘境 |
| Lv.26-45 金丹 | 明确进入中期,需要规划资源 | 10-25 场有效战斗 | 据点、妖巢、宗门任务 |
| Lv.46-70 元婴 | 玩家开始影响地图格局 | 15-35 场有效战斗 | 地盘、Boss、队伍战 |
| Lv.71-100 化神 | 成为势力级角色 | 25-50 场有效战斗 | 大型事件、宗门战争 |
| Lv.101+ 合体以后 | 长线追求,不靠刷小怪推进 | 按章节/赛季目标推进 | 势力战争、天劫、世界事件 |

重点不是强行拖慢,而是让玩家每次升级前都有目标:

```text
我差一点修为就能到筑基。
我还缺一份筑基丹和一处稳定灵脉。
我破境后能开新技能槽、带更多弟子、打之前打不过的妖巢。
```

### 1.2 小等级和大境界分工

| 成长类型 | 频率 | 作用 |
|---|---:|---|
| 小等级 | 高频 | 血量、灵力、少量属性、技能熟练度 |
| 小境界末段 | 中频 | 提升明显,开始卡修为和材料 |
| 大境界突破 | 低频 | 战力跃迁、解锁玩法、地图权限变化 |

小等级可以让玩家持续有反馈,但真正让玩家“爽”的应该是破境。

## 2. 修为口径重做

### 2.1 保留原始修为账本

保留当前 `token` 进入总修为的设计,因为它是《灵枢笔录》的核心特色:

```text
raw_cultivation_total += 本次实际 token 消耗
```

它可以用于:

- 玩家总修行里程碑。
- 宗门名望。
- 个人履历。
- 长线排行榜。
- 特殊称号,例如“万言炼气”“百万灵枢”。

但它不应该 100% 直接推进等级。

### 2.2 新增有效修为

升级应使用有效修为:

```text
effective_exp += 本次有效修为
```

推荐公式:

```text
有效修为 =
  胜利基础修为
+ min(本场 token, 战斗类型 token 上限) * 凝练率
+ 首胜奖励
+ 任务/悬赏奖励
+ 越级挑战奖励
```

推荐参数:

| 战斗类型 | token 计入上限 | 凝练率 | 设计目的 |
|---|---:|---:|---|
| 普通怪 | 1,200 | 60% | 防止长文本普通战斗刷爆等级 |
| 精英怪 | 3,000 | 80% | 鼓励挑战高价值目标 |
| Boss | 8,000 | 100% | Boss 战允许高消耗高回报 |
| 秘境/据点 | 5,000 | 90% | 绑定地图目标和战略收益 |
| 破境事件 | 不按普通战斗算 | 固定结算 | 破境不被 token 长度左右 |

示例:

```text
一场普通怪消耗 6,000 token:
原始修为账本 +6,000
有效修为只按 1,200 * 60% = 720 参与升级
再加胜利基础修为和首胜奖励
```

这样既保留了“我真的消耗了 token”的心理反馈,又不会让普通怪因为一次长文本直接冲破等级。

## 3. 新等级与修为卡点

### 3.1 境界分段

| 大境界 | 等级段 | 设计定位 |
|---|---:|---|
| 炼气 | Lv.1-10 | 新手期,建立基础战斗循环 |
| 筑基 | Lv.11-25 | 正式入门,开始选择流派和队伍 |
| 金丹 | Lv.26-45 | 中期核心,开始争夺资源点 |
| 元婴 | Lv.46-70 | 可独当一面,开启大地图影响 |
| 化神 | Lv.71-100 | 势力级角色,参与宗门战争 |
| 合体 | Lv.101-135 | 宗门核心战力,争夺州域 |
| 大乘 | Lv.136-175 | 世界顶层,准备天劫线 |
| 渡劫 | Lv.176-220 | 高风险突破,失败也应有剧情后果 |
| 飞升 | Lv.221+ | 长线无上限内容,适合赛季和深层玩法 |

### 3.2 单级修为公式

推荐以当前等级 `L` 计算升下一级需求:

```text
升下一级修为 =
  当前境界基础需求 * 当前境界成长系数 ^ 当前境界内序号
+ 如果是境界末级,追加下一境界破境卡点
```

参数建议:

| 境界 | 基础需求 | 单级成长系数 | 破入该境界额外卡点 |
|---|---:|---:|---:|
| 炼气 | 4,500 | 1.16 | 0 |
| 筑基 | 18,000 | 1.12 | 60,000 |
| 金丹 | 52,000 | 1.105 | 180,000 |
| 元婴 | 130,000 | 1.085 | 500,000 |
| 化神 | 300,000 | 1.07 | 1,200,000 |
| 合体 | 700,000 | 1.058 | 3,000,000 |
| 大乘 | 1,500,000 | 1.048 | 7,000,000 |
| 渡劫 | 3,000,000 | 1.04 | 15,000,000 |
| 飞升 | 6,000,000 | 1.035 | 30,000,000 |

破境卡点放在上一境界最后一级,例如:

```text
Lv.10 -> Lv.11 = 炼气末级需求 + 筑基破境卡点
Lv.25 -> Lv.26 = 筑基末级需求 + 金丹破境卡点
```

这会让玩家明确感到:

```text
平时是在升级;
境界末是在冲关。
```

## 4. 血条、蓝条、疲劳的职责

### 4.1 血条

血条代表容错和抗压。

设计目标:

- 小等级每级有稳定提升。
- 大境界突破时提升 40%-65%。
- 不再依赖随机属性决定生死,随机属性只影响 5%-12% 个体差异。

### 4.2 蓝条/灵力

蓝条代表技能循环和爆发次数。

设计目标:

- 前期蓝条不能太大,否则技能消耗无意义。
- 破境后蓝条明显扩大,让玩家能使用更高阶招式。
- 技能消耗建议改为“固定值 + 最大灵力百分比”,避免中后期固定消耗失效。

建议:

```text
普通技能: 8%-12% 最大灵力
高级技能: 16%-24% 最大灵力
绝招: 32%-45% 最大灵力
```

### 4.3 疲劳

疲劳代表每日行动力,不应该像血量一样暴涨。

设计目标:

- 前期疲劳让玩家每天能打 4-6 个目标。
- 中期能打 6-10 个目标。
- 后期提升更多来自丹药、据点、弟子分担,不是角色等级无限堆。

疲劳应慢增长:

```text
炼气 80-95
筑基 100-125
金丹 130-165
元婴 170-215
化神 220-285
```

## 6. 破境收益设计

如果只是修为需求变高,玩家会觉得被拖慢。必须让破境后的收益足够明显。

### 6.1 破境数值跃迁

| 突破 | 血条变化 | 蓝条变化 | 疲劳变化 | 玩家感受 |
|---|---:|---:|---:|---|
| 炼气 -> 筑基 | 260 -> 420 | 160 -> 220 | 95 -> 100 | 终于能稳定用高阶技能 |
| 筑基 -> 金丹 | 760 -> 1,100 | 360 -> 480 | 125 -> 130 | 可以打低阶据点 |
| 金丹 -> 元婴 | 1,900 -> 2,800 | 760 -> 950 | 165 -> 170 | 从个人战变成可带队战 |
| 元婴 -> 化神 | 4,600 -> 6,800 | 1,450 -> 1,850 | 215 -> 220 | 可以参与势力战 |
| 化神 -> 合体 | 10,500 -> 15,000 | 2,800 -> 3,600 | 285 -> 300 | 成为宗门核心战力 |

### 6.2 破境玩法解锁

每次大境界突破都应该给“新东西”,不能只有数字。

| 境界 | 解锁建议 |
|---|---|
| 筑基 | 第二套技能预设、低级法宝槽、悬赏系统 |
| 金丹 | 金丹特性三选一、据点挑战、队伍副将 |
| 元婴 | 元婴化身、弟子协战、灵脉占领 |
| 化神 | 领域技能、宗门战争、州域声望 |
| 合体 | 阵法合击、跨州远征、高阶天材地宝 |
| 大乘 | 天劫线、道统选择、世界事件投票/介入 |
| 渡劫 | 失败后保留残魂路线,成功进入飞升线 |

这样玩家为了破境会有明确欲望:

```text
不是“我要多 200 血”;
而是“我到金丹就能开据点、抢灵脉、带副将”。
```

## 7. 战斗奖励与卡点配合

### 7.1 推荐战斗修为产出

| 目标 | 推荐有效修为 | 推荐用途 |
|---|---:|---|
| 同级普通怪 | 800-1,800 | 日常补差额 |
| 同级精英怪 | 3,000-8,000 | 推进小等级 |
| 首次击败新怪 | 额外 2,000-12,000 | 鼓励图鉴和探索 |
| 小 Boss | 20,000-80,000 | 推境界末段 |
| 大 Boss/据点首通 | 100,000-800,000 | 冲破境卡点 |
| 破境任务 | 固定补足关键缺口 | 防止纯刷怪疲劳 |

按照这套产出:

- Lv.1 -> Lv.2 需要 4,500,玩家大约 3-5 场普通战斗或 1 场精英加任务奖励。
- Lv.5 -> Lv.6 需要 9,500,玩家会开始需要首胜、悬赏或精英怪。
- Lv.10 -> Lv.11 需要 77,000,玩家必须完成筑基任务、低级 Boss 或灵脉节点,不再靠刷小怪。
- Lv.25 -> Lv.26 需要 270,000,玩家需要金丹材料、据点或中型 Boss。

### 7.2 破境材料

修为卡点不应只靠数字。推荐每个大境界需要 2-3 个清晰条件:

| 突破 | 条件 |
|---|---|
| 筑基 | 修为达标 + 筑基丹 + 清理一处低级灵脉 |
| 金丹 | 修为达标 + 金丹灵材三选二 + 击败金丹守关怪 |
| 元婴 | 修为达标 + 元神引 + 占领/协助一处灵脉据点 |
| 化神 | 修为达标 + 领域残章 + 完成势力抉择 |
| 合体 | 修为达标 + 阵法核心 + 宗门战贡献 |

材料不要做成大量碎片,否则会变成刷子疲劳。每个破境最多 2-3 个关键物,每个关键物对应一个玩法目标。

## 8. 对当前代码的落地建议

### 8.1 `backend/app/cultivation.py`

当前 `apply_cultivation_gain()` 同时增加:

```text
cultivation_total
token_total
daily_token_used
monthly_token_used
exp
```

建议改成:

```text
token_total += raw_tokens
raw_cultivation_total += raw_tokens
exp += effective_cultivation_gain(...)
```

短期兼容方案:

```text
cultivation_total 继续表示原始 token 总修为
exp 改为有效修为
```

这样不用立刻迁移所有展示字段。

### 8.2 `backend/app/attributes.py`

需要替换:

```python
next_level_need(current_level)
```

当前:

```text
Lv.1 -> Lv.2 大约只要 200
```

新方案:

```text
Lv.1 -> Lv.2 = 4,500
Lv.10 -> Lv.11 = 77,000
Lv.25 -> Lv.26 = 270,000
Lv.45 -> Lv.46 = 850,000
```

同时 `derive_stats()` 不应继续用过强的 `realm_multiplier(level)` 指数放大。建议改成:

```text
先由等级查目标基础血量/灵力/疲劳;
再由属性给 5%-12% 修正;
装备、心法、境界特性最后乘算。
```

这样玩家数值会稳定,怪物也能围绕同级玩家基准反推。

### 8.3 UI 展示

升级条建议拆成两条:

| 展示 | 作用 |
|---|---|
| 本级有效修为进度 | 告诉玩家离升级还差多少 |
| 总修行 token | 保留产品特色和长期成就 |

破境末级时,升级条文案从:

```text
距离下一级还差 12,000 修为
```

改成:

```text
筑基关: 修为 42,000 / 77,000
尚缺: 筑基丹、低级灵脉净化
突破后: 血量 +62%, 灵力 +38%, 解锁法宝槽
```

玩家需要看到“为什么卡”和“冲过去有什么爽点”。

---

## 9. 验收标准

这套修为曲线落地后,应该满足:

1. 普通怪单场战斗不能让玩家稳定升级。
2. 玩家 10 级前仍然能感到成长,但不会一口气连升多级。
3. 每个大境界末级都形成明确卡点。
4. 破境后血条、蓝条、疲劳和玩法权限都有明显变化。
5. token 消耗仍然被记录和尊重,但不会被 AI 输出长度直接操控等级。
6. 等级、血条、蓝条、疲劳曲线能作为怪物反推基准,后续战斗数值可以稳定围绕它设计。

---

## 10. 最终建议

优先级最高的不是继续加怪物,而是先把成长节奏固定:

```text
第一步: token 总账和有效修为分离。
第二步: 替换 next_level_need。
第三步: 用新表重做 HP/QI/疲劳派生。
第四步: 每个大境界绑定破境材料和玩法解锁。
第五步: 再回头调怪物和 Boss。
```

只有这样,打怪才不会只是“消耗 token 换等级”,而会变成:

```text
我要打这个目标,因为它能帮我突破当前境界。
```
